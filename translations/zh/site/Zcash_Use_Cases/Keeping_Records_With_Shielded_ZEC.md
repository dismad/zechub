# 使用屏蔽 ZEC 保存记录

## TL;DR

- 屏蔽资金是私密的，但你仍然可以保存清晰、完整的财务记录
- 备注就像你账本中的条目，例如发票编号或付款用途
- Viewing Key 允许你，或你选择的人（例如会计），在不公开历史记录的情况下查看你的记录
- 你可以汇总任意时间段的收入和支出，这正是报表或报税所需要的
- 这一切都不会削弱你的隐私，因为由你决定谁能看到什么

<br/>

## 这篇内容适合谁？

- 以 ZEC 收款的自由职业者和小型企业
- 任何需要在保持隐私的同时记账的人
- 正在为会计或报税准备记录的人

<br/>

## 挑战

隐私和记账看起来似乎是相互对立的。如果你的交易是屏蔽的，金额和地址对公众都是隐藏的，那么你该如何妥善记账，或者向会计证明你的收入呢？

对于 Zcash 来说，这是一种错误的取舍。屏蔽交易默认会对所有人隐藏你的活动，但 Zcash 也为你提供了工具，让你可以按照自己的条件，向真正需要这些记录的人披露你自己的账目。你可以同时对世界保持私密，又对你的会计保持透明。

<br/>

## 备注就是你的账本

每一笔屏蔽交易（z 到 z）都可以附带一条加密的[备注](/using-zcash/memos)。对于记账来说，备注就是你写明付款用途的地方：发票编号、客户名称、项目代码，或者像“3月房租”这样的简短说明。

因为备注会随交易一起发送，并且只有相关方可以读取，它就成为你账簿中的私密条目。当你或你的客户在每笔付款中都附上清晰的备注时，你的交易历史就会从一串没有上下文的金额列表，变成可实际使用的账本。

一个简单的习惯：与客户约定始终在备注中填写发票编号。之后，将付款与发票对应起来就会非常直接。

<br/>

## 查看你自己的历史记录

要记账，你需要能看到自己的活动。你的钱包持有用于解密屏蔽交易的密钥，因此钱包可以向你展示完整信息：日期、金额、哪些是收入、哪些是支出，以及附带的备注。

这是公众看不到、但你可以看到的部分，因为这些数据属于你。定期查看你的历史记录，而不是等到年底再处理，可以让记录更准确，也更容易发现错误。

<br/>

## 与会计共享记录

当你需要让他人查看你的屏蔽活动时，例如会计或审计人员，你不必交出你的支出密钥，也不需要将任何内容公开。你只需要共享一个[Viewing Key](/zcash-tech/viewing-keys)。

完整的 Viewing Key 是只读的。它可以让持有者查看某个地址的收入和支出交易，包括金额和备注，但绝不会让他们动用你的资金。这使它成为提供给会计的安全选择。他们获得了所需的可见性，你的钱仍由你掌控，而世界上的其他人依然什么也看不到。

这被称为选择性披露，也是屏蔽 Zcash 能够支持诚实记账而不是妨碍记账的实际原因之一。

<br/>

## 汇总某一时间段的数据

对于大多数报表工作，你需要统计某一时间范围内的总额：这个季度收到了多少、支出了多少、净头寸是多少。既然你可以查看自己的完整历史记录，你就可以按自己选择的任何时间段来汇总这些数据，例如一个月、一个季度或一年。

保持备注的一致性会让这件事更容易，因为你可以按付款用途分组，而不仅仅是按日期和金额分组。

<br/>

## 关于税务的说明

各国税务规则不同，而且会随着时间变化，因此这里提供的是一般信息，而不是税务建议。在许多地方，接收或处置加密货币都可能带来税务后果，并且你可能需要保留相关记录，包括你收到的内容、时间，以及当时的价值。

好消息是，屏蔽 Zcash 不会妨碍你履行这些义务。你可以保存完整的私密记录，按税务机关要求的期间进行汇总，并通过 Viewing Key 向会计或税务机关披露这些记录，而不必将你的活动公开。如果你不确定自己的义务是什么，请咨询你所在国家的合格专业人士。

<br/>

## 需要避免的常见错误

- 不写备注，导致年底只剩下一串金额而没有上下文
- 所有用途都重复使用同一个地址，这会让区分客户或用途变得更困难
- 一直等到报税季才回看一整年的历史，而不是在过程中持续记账
- 在会计只需要只读 Viewing Key 的情况下，错误地共享了支出密钥
