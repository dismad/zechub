# Размещение репозитория GitHub через IPFS

## Введение

В этом руководстве вы узнаете, как создать URL-адрес, который можно клонировать с помощью Git для вашего репозитория на GitHub, размещённого с использованием CID IPFS.

Это полезно для обеспечения доступности контента независимо от географического региона, сопротивления цензуре и как постоянной резервной копии важной информации!

Примечание: данные, загруженные в IPFS, доступны всем пользователям сети. Вы можете захотеть локально шифровать персональные/конфиденциальные данные.

## Установка IPFS Kubo

Следуйте инструкциям по установке, предоставленным [здесь](https://docs.ipfs.tech/install/command-line/#install-official-binary-distributions)

В этом примере мы используем Linux, доступны и другие версии ОС.

Проверьте успешность установки с помощью команды ipfs –version

## Клонирование репозитория

Чтобы начать, выберите репозиторий Git, который хотите разместить и скопируйте его:

Выполните команду: “git clone https://github.com/zechub/zechub”

![https://i.ibb.co/HxFX37b/Screenshot-from-2023-05-20-14-14-46.png](https://i.ibb.co/HxFX37b/Screenshot-from-2023-05-20-14-14-46.png)

Теперь подготовьте его к клонированию через IPFS.

cd zechub git update-server-info

Распакуйте объекты Git:

![](https://i.ibb.co/25RwyWz/image-2024-04-20-175848513.png)

Выполнение этого позволит IPFS дедуплицировать объекты, если вы обновите репозиторий Git позже.

## Добавление в IPFS

После выполнения вышеуказанных действий, репозиторий готов к размещению. Остается только добавить его в IPFS:

$ pwd

/code/myrepo

$ ipfs add -r

## Клонирование через IPFS

Теперь вы должны иметь возможность получить доступ к репозиторию GitHub, используя:

git clone http://ipfs.io/ipfs/yourCID

В качестве альтернативы вы можете искать и получать данные с помощью вашего локального узла IPFS.

Последнее примечание: папка репозитория на IPFS не обновляется вместе с реальным репозиторием GitHub. Рекомендуется перезагружать папку регулярно.
