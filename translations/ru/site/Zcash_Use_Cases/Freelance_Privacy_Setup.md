## TL;DR

- Получайте платежи через защищенные адреса
- Разделяйте идентичности по клиентам (если это необходимо)
- Используйте мемо для внутреннего отслеживания
- Сохраняйте приватность доходов

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="step icon"/> Пошаговое руководство

### Шаг 1: Создайте адрес для получения платежа

Создайте защищенный (z) адрес.

Опционально:
- Используйте разные адреса для разных клиентов


### Шаг 2: Поделитесь адресом с клиентом

Отправьте свой z-адрес напрямую клиенту.

Избегайте:
- Публикации его, если это не требуется


### Шаг 3: Используйте мемо для отслеживания

При получении платежей:
- Попросите клиентов включить мемо (например, идентификатор счета)

Это поможет вам:
- Отслеживать платежи без публичного раскрытия данных


### Шаг 4: Сохраняйте разделение идентичностей

Если это необходимо:
- Используйте отдельные кошельки или адреса для каждого клиента
- Избегайте связывания идентичностей между собой


<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-cancel.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="cancel icon"/> Ошибки, которых стоит избегать

- Использование прозрачных адресов для получения платежей  
- Повторное использование одного и того же адреса для всех клиентов  
- Публичная связь кошелька с идентичностью  
- Непрерывный отслеживание платежей (приводит к путанице)
