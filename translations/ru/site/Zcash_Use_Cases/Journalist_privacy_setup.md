## TL;DR

- Используйте защищенные адреса для всех транзакций
- Никогда не раскрывайте идентификатор кошелька публично
- Защитите источники, избегая отслеживаемых платежей
- Осторожно используйте мемо для безопасной коммуникации

- Кто заплатил вам
- Кому вы заплатили
- Когда и сколько

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="иконка шага"/> Пошаговое руководство

### Шаг 1: Создайте отдельный кошелек

- Используйте кошелек только для чувствительной работы
- Не смешивайте с личными транзакциями

### Шаг 2: Используйте только защищенные адреса

- Всегда используйте z-адреса для транзакций
- Повсеместно избегайте прозрачных транзакций

### Шаг 3: Защитите платежи источникам

Если вы платите источникам:
- Используйте защищенные транзакции (z → z)
- Избегайте предсказуемых временных паттернов
- Не повторяйте адреса

### Шаг 4: Осторожно используйте мемо

- НЕ включайте чувствительную информацию в мемо
- Используйте нейтральные или зашифрованные ссылки при необходимости

### Шаг 5: Сохраняйте операционную безопасность

- Избегайте связи кошелька с вашей идентичностью
- Не подтверждайте транзакции публично
- Используйте отдельные каналы для коммуникации
