## TL;DR

- Используйте защищенные адреса для хранения общих средств
- Ограничьте видимость балансов и транзакций
- Разделите операционные роли от публичной идентичности
- Используйте заметки для внутренней ответственности

Фонды сообществ в публичных блокчейнах раскрывают:
- Общий баланс казны
- Историю трат
- Идентичности участников

Это создает риски:
- Целевые атаки на средства
- Внутренние конфликты из-за давления прозрачности
- Потеря стратегической приватности

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-lock.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="toolbox icon"/> Почему Zcash?

Zcash позволяет сообществам:
- Хранить средства приватно
- Выполнять транзакции без раскрытия деталей
- Сохранять внутреннюю ответственность, не раскрываясь публично

### Шаг 1: Создайте защищенный адрес казны

- Сгенерируйте **z-адрес** для казны
- Это станет основным счетом для хранения средств


### Шаг 2: Определите роли

Разделите обязанности:
- Кассир (управляет средствами)
- Участники (запрашивают расходы)
- Рецензенты (одобряют внутри)

Избегайте связывания этих ролей с публичными идентичностями.



### Шаг 3: Получайте средства приватно

- Принимайте взносы через защищенный адрес
- Поощряйте участников отправлять средства из защищенных кошельков



### Шаг 4: Отслеживайте транзакции внутри

Используйте:
- Заметки в кошельке
- Внешние журналы (например, общие документы)

Пример заметки: 
 - Грант #12 - Оплата разработчика



### Шаг 5: Выполняйте платежи приватно

- Переводите средства через защищенные транзакции (z → z)
- Избегайте раскрытия идентичности получателей публично
