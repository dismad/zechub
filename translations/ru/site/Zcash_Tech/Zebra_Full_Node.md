## Введение в узел Zebra

Представляем Zebra: революция в инфраструктуре узлов Zcash на Rust

Познакомьтесь с Zebra — выдающимся достижением, первым узлом Zcash, полностью созданным на Rust. Полностью интегрированный в одноранговую сеть Zcash, Zebra служит ключевым инструментом, повышающим устойчивость сети. Выполняя основные функции по проверке и ретрансляции транзакций, а также тщательно поддерживая состояние блокчейна Zcash, Zebra способствует созданию более децентрализованной сетевой инфраструктуры.

## Преимущества по сравнению с реализацией узла Zcashd
В отличие от исходного узла Zcash — zcashd, который восходит к базовому коду Bitcoin и разрабатывается Electric Coin Company, наша реализация представляет собой автономное решение. Созданный с нуля с акцентом на безопасность и эффективность, Zebra использует возможности Rust — языка с безопасным управлением памятью.

Несмотря на различное происхождение, и zcashd, и Zebra следуют одному и тому же протоколу, обеспечивая бесшовную коммуникацию и совместимость между ними. Эта инновация не только расширяет экосистему Zcash, но и задаёт новый стандарт для разработки блокчейн-узлов.

## Инструкции по запуску Zebra Launcher

Вы можете запустить Zebra с помощью нашего Docker-образа или собрать его вручную. Пожалуйста, ознакомьтесь с разделом «Системные требования».

### Использование Docker:

Чтобы без труда запустить наш последний релиз и синхронизировать его с актуальным состоянием сети, выполните следующую команду:

```

docker run zfnd/zebra:latest

```

Более подробные инструкции и детальную информацию можно найти в нашей [документации по Docker](https://zebra.zfnd.org/user/docker.html).

### Сборка Zebra:

Для сборки Zebra требуются Rust, libclang и компилятор C++.

- Убедитесь, что у вас установлена последняя стабильная версия Rust, поскольку Zebra тестируется исключительно с ней.
- Необходимые зависимости для сборки включают:
  - libclang (также известный как libclang-dev или llvm-dev)
  - clang или другой компилятор C++ (например, g++ для всех платформ или Xcode для macOS)
  - protoc (компилятор Protocol Buffers) с флагом *--experimental_allow_proto3_optional*, введённым в Protocol Buffers v3.12.0 (выпущена 16 мая 2020 года).



### Зависимости в Arch:

После того как вы убедились, что все зависимости установлены, перейдите к сборке и установке Zebra с помощью следующей команды:

```

cargo install --locked zebrad

```

Запустите Zebra, выполнив:

```
zebrad start

```


## Дополнительные настройки и возможности:


### - Инициализация файла конфигурации:

  - Сгенерируйте файл конфигурации с помощью команды:
    
  ```
  zebrad generate -o ~/.config/zebrad.toml
  
  ```

  - Сгенерированный файл *zebrad.toml* будет помещён в стандартный каталог настроек Linux. Альтернативные пути по умолчанию для других ОС см. в нашей документации.



### - Настройка индикаторов прогресса:

  - Настройте *tracing.progress_bar* в вашем *zebrad.toml*, чтобы отображать важные метрики в терминале с помощью индикаторов прогресса. Примечание: существует известная проблема, из-за которой оценки в индикаторах прогресса могут становиться чрезмерно большими.



### - Настройка майнинга:

  - Zebra можно настроить для майнинга, указав *MINER_ADDRESS* и сопоставление порта в Docker. Подробности см. в нашей [документации по поддержке майнинга](https://zebra.zfnd.org/user/mining-docker.html).


### - Пользовательские функции сборки:

  - Расширяйте функциональность Zebra с помощью дополнительных возможностей Cargo, таких как метрики Prometheus, мониторинг Sentry, экспериментальная поддержка Elasticsearch и другие.

  - Вы можете объединять несколько возможностей, перечисляя их как параметры флага `--features` во время установки.


### Примечание: Некоторые функции отладки и мониторинга отключены в релизных сборках для оптимизации производительности.

Полный список экспериментальных и ориентированных на разработчиков возможностей см. в нашей [документации API](https://docs.rs/zebrad/latest/zebrad/index.html#zebra-feature-flags).
 

# Системные требования и настройка сети для Zebra

Чтобы обеспечить оптимальную производительность и надёжность, мы рекомендуем следующие системные требования для компиляции и запуска zebrad — революционного узла Zcash, полностью созданного на Rust:

### Системные требования:
- CPU: 4 ядра CPU
- RAM: 16 ГБ
- Дисковое пространство: 300 ГБ свободного места на диске для компиляции бинарных файлов и хранения кэшированного состояния цепочки
- Сеть: сетевое подключение 100 Мбит/с с минимум 300 ГБ входящего и исходящего трафика в месяц


Обратите внимание, что выполнение набора тестов Zebra может занять более часа в зависимости от характеристик вашего устройства. Хотя более медленные системы, возможно, смогут компилировать и запускать Zebra, точные пределы производительности пока не определены в ходе тестирования.


### Требования к диску:
- Zebra использует примерно 300 ГБ для кэшированных данных Mainnet и 10 ГБ для кэшированных данных Testnet. Ожидайте, что использование диска со временем будет расти.
- База данных регулярно очищается, особенно во время выключения или перезапуска, что обеспечивает целостность данных. Незавершённые изменения из-за принудительного завершения или паники откатываются при перезапуске Zebra.


### Сетевые требования и порты:
- Zebra использует следующие TCP-порты для входящих и исходящих соединений:
  - 8233 для Mainnet
  - 18233 для Testnet
- Настройка Zebra с определённым listen_addr позволяет объявлять этот адрес для входящих соединений. Хотя исходящие соединения необходимы для синхронизации, входящие соединения являются необязательными.
- Необходим доступ к DNS seeders сети Zcash через системный DNS-резолвер ОС (обычно порт 53).
- Хотя Zebra может устанавливать исходящие соединения на любом порту, zcashd предпочитает узлы на портах по умолчанию, чтобы снизить риск DDoS-атак на другие сети.


### Типичное использование сети в Mainnet:
- Первичная синхронизация: для начальной синхронизации требуется загрузка 300 ГБ, причём ожидается рост объёма последующих загрузок.
- Текущие обновления: ожидайте ежедневный входящий и исходящий трафик в диапазоне от 10 МБ до 10 ГБ в зависимости от размера пользовательских транзакций и запросов от других узлов.
- Zebra выполняет начальную синхронизацию при каждом изменении внутренней версии базы данных, что потенциально может потребовать полной загрузки цепочки при обновлении версии.
- Предпочтительны узлы с задержкой в обе стороны не более 2 секунд. Если задержка превышает этот порог, пожалуйста, отправьте тикет для получения помощи.
