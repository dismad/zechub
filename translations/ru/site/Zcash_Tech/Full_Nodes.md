# Полные узлы

Полный узел — это программное обеспечение, которое запускает полную копию блокчейна любой криптовалюты и предоставляет доступ к функциям протоколов.

Он хранит полную запись каждой транзакции, которая произошла с момента создания (с момента генезиса), поэтому он может проверять достоверность новых транзакций и блоков, добавляемых в блокчейн.

## Zcashd

Zcashd — это основная реализация полного узла, используемая Zcash. Она разработана и поддерживается Electric Coin Company.

Zcashd предоставляет набор API через свой интерфейс RPC. Эти API предоставляют функции, позволяющие внешним приложениям взаимодействовать с узлом.

[Lightwalletd](https://github.com/zcash/lightwalletd) — это пример приложения, использующего полный узел для того, чтобы разработчики могли создавать и поддерживать мобильные приватные легкие кошельки без необходимости напрямую взаимодействовать с Zcashd.

[Полный список поддерживаемых команд RPC](https://zcash.github.io/rpc/)

[Книга по Zcashd](https://zcash.github.io/zcash/)


### Запуск узла (Linux)

- Установите зависимости

      sudo apt update

      sudo apt-get install \
      build-essential pkg-config libc6-dev m4 g++-multilib \
      autoconf libtool ncurses-dev unzip git python3 python3-zmq \
      zlib1g-dev curl bsdmainutils automake libtinfo5

- Клонируйте последний релиз, выполните checkout, настройку и сборку:

      git clone https://github.com/zcash/zcash.git

      cd zcash/

      git checkout v5.4.1
      ./zcutil/fetch-params.sh
      ./zcutil/clean.sh
      ./zcutil/build.sh -j$(nproc)

- Синхронизация блокчейна (может занять несколько часов)

    Чтобы запустить узел, выполните:

      ./src/zcashd

- Приватные ключи хранятся в ~/.zcash/wallet.dat

[Руководство по установке Zcashd на Raspberry Pi](https://zechub.notion.site/Raspberry-Pi-4-a-zcashd-full-node-guide-6db67f686e8d4b0db6047e169eed51d1)


## Zebra

Zebra — это независимая реализация полного узла для протокола Zcash, созданная Zcash Foundation.

На данный момент она проходит тестирование и всё ещё экспериментальная.

У Zebra есть два основных компонента. Компонент-клиент отвечает за сканирование блокчейна и попытку дешифрования транзакций.

Вторая часть — это утилита командной строки zebra. Эта утилита управляет ключами расхода, адресами и взаимодействует с компонентом-клиентом в zebrad для предоставления базовых функций кошелька.

Любой, кто хочет попробовать Zebra для добычи блоков, приглашается присоединиться к серверу R&D Discord. Также обязательно прочтите книгу по Zebra для получения инструкций по настройке.

[Github](https://github.com/ZcashFoundation/zebra/)

[Книга по Zebra](https://zebra.zfnd.org) 

[Discord](https://discord.gg/uvEdHsrb)



## Сеть

Запуская полный узел, вы помогаете укрепить сеть zcash, поддерживая её децентрализацию.

Это помогает предотвратить контроль со стороны врагов и сохранить сеть устойчивой к некоторым формам нарушений.

DNS-сеедеры предоставляют список других надежных узлов через встроенный сервер. Это позволяет транзакциям распространяться по всей сети.

### Статистика сети

Это примеры платформ, которые позволяют получить доступ к данным сети Zcash:

[Обозреватель блоков Zcash](https://zcashblockexplorer.com)

[Coinmetrics](https://docs.coinmetrics.io/info/assets/zec)

[Blockchair](https://blockchair.com/zcash)

Вы также можете внести свой вклад в развитие сети, запуская тесты или предлагая новые улучшения и предоставляя метрики. 



### Добыча

Майнерам требуются полные узлы для доступа ко всем RPC-интерфейсам, связанным с добычей, таким как getblocktemplate и getmininginfo.

Zcashd также позволяет добывать приватные монеты. Майнеры и пулы добычи могут выбирать возможность добывать напрямую для накопления приватных ZEC в z-адресе по умолчанию.

Прочтите [Руководство по добыче](https://zcash.readthedocs.io/en/latest/rtd_pages/zcash_mining_guide.html) или присоединитесь к странице сообщества для [Майнеров Zcash](https://forum.zcashcommunity.com/c/mining/13).

### Приватность 

Запуск полного узла позволяет вам самостоятельно проверять все транзакции и блоки в сети Zcash.

Запуск полного узла избегает некоторых рисков для приватности, связанных с использованием сторонних служб для проверки транзакций от вашего имени.

Использование своего собственного узла также позволяет подключаться к сети через [Tor](https://zcash.github.io/zcash/user/tor.html).
Это имеет дополнительное преимущество, позволяя другим пользователям приватно подключаться к вашему узлу по адресу .onion.
