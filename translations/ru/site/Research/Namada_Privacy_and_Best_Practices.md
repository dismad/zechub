# Лучшие практики конфиденциальности в Namada

> Практическое руководство с конкретными рекомендациями для достижения максимальной конфиденциальности в Namada — и точного понимания того, где заканчивается её защита.

**Конфиденциальность — это фундаментальное право.** Namada была специально создана для его защиты с помощью передовой криптографии с нулевым разглашением. Это руководство объединяет наиболее эффективные практики, используемые пользователями и разработчиками, для которых конфиденциальность имеет первостепенное значение.

---

## Как Namada защищает вашу конфиденциальность

Namada — это суверенный блокчейн, ориентированный прежде всего на конфиденциальность, который скрывает адреса кошельков, суммы транзакций и балансы с помощью **доказательств с нулевым разглашением (zk-SNARKs)**.

### Основные функции конфиденциальности

- **Экранированные транзакции** - полностью скрывают отправителя, получателя и суммы.
- **Multi-Asset Shielded Pool (MASP)** - приватные переводы, обмены и мосты для любых активов.
- **Межсетeвая конфиденциальность** - экранированные мосты через IBC (поддержка Ethereum и Solana появится в ближайшее время).
- **Вознаграждения за экранированную доходность** - получайте токены NAM, просто используя экранированные транзакции.
- **Низкие комиссии** - высокий уровень конфиденциальности без ущерба для удобства использования.

## Важные ограничения

Даже самая сильная ончейн-конфиденциальность может быть подорвана поведением пользователя или офчейн-факторами.

**Namada НЕ защищает от:**

- Подключения без VPN или Tor (ваш IP-адрес раскрывается)
- Повторного использования экранированных адресов
- Выполнения прозрачных (неэкранированных) транзакций
- Привязки вашего адреса Namada к социальным сетям или реальной личности
- Использования централизованных KYC-бирж для депозитов или вывода средств

## Лучшие практики для максимальной конфиденциальности

### 1. Общие принципы
- По умолчанию используйте **экранированные транзакции** для каждого действия.
- Никогда не используйте одни и те же экранированные адреса для разных целей.
- Избегайте смешивания экранированной и прозрачной активности в рамках одной сессии.

### 2. Мосты активов
- Используйте выделенный прозрачный адрес **только** для входящих мостов.
- Сразу экранируйте активы после перевода через мост в Namada.
- По возможности минимизируйте вывод активов из Namada через мосты.

### 3. MASP (Multi-Asset Shielded Pool)
- По умолчанию держите все активы внутри MASP.
- Рассматривайте свой баланс MASP как основной приватный кошелёк.

### 4. Ключи просмотра
- Делитесь viewing keys **только** с теми, кому вы полностью доверяете.
- Никогда не публикуйте и не размещайте viewing keys в открытом доступе.

### 5. Гигиена транзакций
- Варьируйте время и суммы между транзакциями.
- По возможности объединяйте несколько транзакций в пакет.
- Избегайте отправки круглых или легко распознаваемых сумм.

### 6. Операционная безопасность
- Всегда используйте **VPN** (в идеале Tor) при взаимодействии с кошельками или dApps.
- Никогда не делитесь скриншотами, содержащими адреса или балансы.
- Используйте отдельные кошельки для разных видов деятельности (трейдинг, пожертвования, личное использование).

## Расширенный контрольный список по конфиденциальности

1. **Всегда сначала экранируйте** - переводите активы в MASP перед проведением транзакций.
2. **Регулярно меняйте экранированные адреса** для разных сценариев использования.
3. **Выводите средства напрямую на экранированные адреса** с бирж, когда это возможно.
4. **Варьируйте время транзакций**, чтобы разрушить распознаваемые шаблоны.
5. **Используйте аппаратные кошельки** для крупных сумм.
6. **Обновляйте программное обеспечение** - всегда используйте последнюю версию клиента Namada.
7. **Защитите своё устройство** с помощью надёжного шифрования и менеджеров паролей.
8. **Будьте предельно осторожны** с утечками метаданных в чатах или публичных логах.
