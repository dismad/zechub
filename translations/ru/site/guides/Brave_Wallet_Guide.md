# Как отправлять и получать защищённый Zcash (ZEC) с помощью Brave Wallet: пошаговое руководство по приватности

Brave Wallet — это безопасный нативный криптокошелёк, встроенный непосредственно в браузер Brave, без необходимости устанавливать расширения.


## Почему поддержка защищённого ZEC имеет значение


Zcash — одна из немногих криптовалют, предлагающих приватность по умолчанию благодаря защищённым транзакциям.

Brave Wallet теперь поддерживает защищённый ZEC Orchard, позволяя выполнять приватные транзакции без стороннего ПО.

Теперь пользователи могут отправлять и получать ZEC, не раскрывая балансы или метаданные, — и всё это прямо в браузере Brave.

## Часть 1: Настройка Brave Wallet

**Рассматриваемые шаги:**

1. Откройте браузер Brave (пока только на компьютере)

2. Нажмите на значок Wallet или перейдите по адресу brave://wallet

3. Настройте свой кошелёк (создайте новый или импортируйте существующий)

4. Откройте криптовалютную панель управления


![img1](https://github.com/user-attachments/assets/f54cd1a1-8569-4925-ba1c-7597d030593e)


## Часть 2: Добавление Zcash (ZEC) в Brave Wallet

**Рассматриваемые шаги:**

1. На панели Wallet нажмите "Manage Assets"

2. Найдите ZEC и включите его

3. Теперь ZEC отображается в вашем кошельке


![img2](https://github.com/user-attachments/assets/6f2b2190-cf55-4394-9d5f-29ff9b5bb525)


## Часть 3: Получение защищённого ZEC (Unified Address)

**Рассматриваемые шаги:**

1. Нажмите на ZEC в списке активов

2. Выберите "Receive"

3. Brave сгенерирует Unified Address (UA) — один адрес, который поддерживает как защищённые, так и прозрачные транзакции

4. Скопируйте адрес или отсканируйте QR-код

5. Поделитесь своим UA с отправителем



![img3](https://github.com/user-attachments/assets/53c940b6-1a03-4fa7-aefa-d3478f678a88)



## Часть 4: Приватная отправка защищённого ZEC

**Рассматриваемые шаги:**

1. Выберите ZEC -> Send

2. Вставьте Unified Address получателя или защищённый адрес

3. Введите сумму

4. Подтвердите детали транзакции

5. Нажмите Send — средства будут переведены приватно с использованием пула Orchard


![img4](https://raw.githubusercontent.com/Kellyjoe8/zechub/refs/heads/main/Internet_20250808_172118_4.webp)




## Типы адресов

Вот объяснение каждого типа адресов Zcash:

**t-addr (t1...)** — Прозрачные адреса работают как адреса Bitcoin, а значит, транзакции являются публичными в блокчейне. Их проще использовать с кошельками, не поддерживающими Zcash, но они не обеспечивают приватность.

**z-addr (zs...)** — Защищённые адреса используют технологии приватности Zcash (Sprout/Sapling), чтобы скрывать отправителя, получателя и сумму. Раскрыть детали могут только ключи просмотра или владелец.

**UA (u... or QR)** — Unified Addresses объединяют несколько типов адресов (прозрачные, защищённые, Orchard) в одном. Это обеспечивает максимальную совместимость, при этом по возможности отдавая приоритет приватности.
