# Развёртывание zcashd в Akash через Console

## Что вы развёртываете

Полный узел zcashd, который будет:

-> Синхронизировать весь блокчейн Zcash (350GB+ для mainnet, ~ 40GB для testnet)

-> Стоить примерно $15/месяц в зависимости от цены токена AKT

-> Полностью синхронизироваться от нескольких часов до нескольких дней

-> Использовать 4 vCPU, 16GB RAM, 350GB хранилища (mainnet) или 2 vCPU, 8GB RAM, 50GB (testnet)

-> Загружать криптографические параметры при первом запуске (~ 2GB, однократно)

**zcashd vs Zebra:**

-> zcashd — это оригинальная реализация узла Zcash от Electric Coin Company

-> Zebra — это альтернативная реализация от Zcash Foundation

-> Обе совместимы с сетью Zcash

-> У zcashd больше возможностей (майнинг, кошелёк, API Insight Explorer)

-> Используйте zcashd, если вам нужна функциональность кошелька или специфические RPC API


### **Важно: сопоставление портов в Akash**

Когда вы открываете порт в Akash (например, порт 8233 для P2P zcashd), он **НЕ привязывается к этому же порту** на публичном IP провайдера. Вместо этого провайдер назначает случайный высокий порт (например, 31234 или 42567) и настраивает reverse proxy на порт 8233 вашего контейнера.

Так задумано — провайдеры запускают несколько развёртываний, и возникли бы конфликты, если бы все пытались использовать порт 8233 напрямую.

**Что это значит для вас:**

-> Вы настраиваете порт 8233 в SDL (стандартный P2P-порт zcashd)

-> Akash выдаёт вам URI вроде *provider.com:31234*

-> Другие узлы Zcash подключаются к вам по адресу *provider.com:31234*

-> Внутри контейнера zcashd по-прежнему слушает на 8233


Это обрабатывается автоматически. Просто используйте URI, который выдаёт Akash.

## Предварительные требования

-> Установлено расширение браузера **Keplr Wallet** (Chrome/Brave/Firefox)

-> **Токены AKT** — получите 50-100 AKT на бирже (Coinbase, Kraken, Osmosis)

-> **5 минут**, чтобы пройти через UI Console


## Шаг 1: Подключите ваш кошелёк

-> Перейдите на [https://console.akash.network](https://console.akash.network)

-> Нажмите **"Connect Wallet"** в правом верхнем углу

-> Выберите **Keplr** (или предпочитаемый вами Cosmos-кошелёк)

-> Подтвердите подключение, когда появится окно Keplr


Ваш баланс AKT должен появиться в правом верхнем углу. Если он нулевой, сначала пополните кошелёк.

## Шаг 2: Создайте развёртывание

-> Нажмите кнопку **"Deploy"** (большая синяя кнопка в центре страницы)

-> Выберите **"Build your template"** (или сразу перейдите к загрузке SDL)

### Вариант A: Загрузить файл SDL (рекомендуется)

[![Развернуть в Akash](https://raw.githubusercontent.com/akash-network/console/refs/heads/main/apps/deploy-web/public/images/deploy-with-akash-btn.svg)](https://console.akash.network/new-deployment?step=edit-deployment&templateId=akash-network-awesome-akash-zcash-zcashd)

### Вариант B: Использовать редактор SDL

Если вы хотите вручную вставить SDL:

-> Скопируйте содержимое *zcashd-akash.yml*

-> Вставьте его в редактор SDL

-> Измените по необходимости (см. раздел конфигурации ниже)

-> Нажмите **"Create Deployment"**


## Шаг 3: Проверьте и подтвердите депозит

Console покажет вам:

-> **Депозит развёртывания**: ~ 5 AKT (вы получите его обратно, когда закроете развёртывание)

-> **Оценочная стоимость**: на основе цен, указанных в вашем SDL


Нажмите **"Approve"** и подпишите транзакцию в Keplr.

## Шаг 4: Выберите провайдера

Примерно через ~ 30 секунд вы увидите ставки от провайдеров. Каждая ставка показывает:

-> **Цену за блок** (в AKT или USDC)

-> **Оценочную стоимость в месяц**

-> **Информацию о провайдере** (аптайм, регион и т. д.)


**Не выбирайте просто самый дешёвый вариант.** Проверьте:

-> % аптайма (стремитесь к > 95%)

-> Регион (чем ближе к вам, тем лучше задержка, но для узлов блокчейна это не так важно)

-> Статус аудита (зелёная галочка = выше доверие)


Нажмите **"Accept Bid"** у выбранного провайдера и подпишите в Keplr.

## Шаг 5: Дождитесь развёртывания

Console выполнит следующее:

-> Создаст lease с выбранным провайдером

-> Отправит manifest (он сообщает провайдеру, что запускать)

-> Запустит ваш контейнер


Это занимает 1-2 минуты. В UI вы увидите обновления статуса.

## Шаг 6: Убедитесь, что всё запущено

После развёртывания вы увидите:

-> Вкладку **Services**: показывает ваш сервис *zcashd* и его статус

-> Вкладку **Logs**: журналы в реальном времени от вашего узла zcashd

-> Вкладку **Leases**: сведения о вашем развёртывании (DSEQ, провайдер, стоимость)


### Проверьте журналы

Нажмите **Logs**, и вы должны увидеть запуск zcashd:

```bash
[zcashd]: ZCASHD_NETWORK=mainnet
[zcashd]: Starting: zcashd -printtoconsole -showmetrics=1
...
```

**При первом запуске будут загружены zcash-params (~2GB).** Это одноразовая операция, которая занимает 5-10 минут в зависимости от пропускной способности провайдера. При последующих перезапусках этот шаг будет пропущен.

Синхронизация займёт **от часов до дней** в зависимости от сети. Следите за следующим:

-> Рост высоты блока

-> Подключения к пирам (должно быть 10-30 пиров)

-> Отсутствие повторяющихся ошибок


## Шаг 7: Получите адрес вашего узла

Перейдите на вкладку **Leases**, затем **URIs**.

Вы увидите что-то вроде:

```
zcashd-8233: provider-hostname.com:31234
```

Это ваш **публичный P2P-эндпоинт** узла. Другие узлы Zcash будут подключаться к вам по этому адресу.

**Обратите внимание на сопоставление портов:** вы настроили порт 8233 в SDL, но Akash назначил ему другой публичный порт (31234 в этом примере). Это нормально — см. раздел "Port Mapping on Akash" выше, если это вас смущает. Ваш узел доступен на том порту, который показывает Akash, а не обязательно на 8233.

Если вы включили RPC (по умолчанию он закомментирован в SDL), вы также увидите здесь RPC-эндпоинт с его собственным сопоставленным портом.

## Варианты конфигурации

### Переключение на Testnet

По умолчанию SDL использует Mainnet. Чтобы использовать Testnet:

-> **Измените сеть в разделе *env*:**

   ```yaml
   # - "ZCASHD_NETWORK=mainnet"
   - "ZCASHD_NETWORK=testnet"
   ```

-> **Обновите открываемый порт** в разделе *expose*:

   ```yaml
   # Comment out Mainnet port:
   # - port: 8233
   #   as: 8233
   #   to:
   #     - global: true
   #   proto: tcp

   # Uncomment Testnet port:
   - port: 18233
     as: 18233
     to:
       - global: true
     proto: tcp
   ```

-> **Необязательно: уменьшите ресурсы** для Testnet в *profiles.compute.zcashd.resources*:

   ```yaml
   cpu:
     units: 2  # Down from 4
   memory:
     size: 8Gi  # Down from 16Gi
   storage:
     - size: 50Gi  # Down from 150Gi
   ```

-> **Необязательно: снизьте цену** в *profiles.placement.akash.pricing*:

   ```yaml
   amount: 5000  # Down from 10000
   ```

> обратите внимание: снижение цены может отфильтровать наших провайдеров от участия в торгах. поэкспериментируйте с этим значением или используйте endpiont провайдера, чтобы проверить, сделали бы они ставку. (см. документацию по api провайдера)

### Включение RPC-доступа

По умолчанию RPC отключён из соображений безопасности. Чтобы включить его:

**КРИТИЧНО: задайте надёжные учётные данные.** RPC в zcashd передаёт имя пользователя/пароль по HTTP (не HTTPS). Открывайте RPC наружу только если понимаете последствия для безопасности.

-> Раскомментируйте в разделе *env*:

   ```yaml
   - "ZCASHD_RPCUSER=yourusername"
   - "ZCASHD_RPCPASSWORD=your_very_strong_password_here"  # Use a real password
   - "ZCASHD_RPCBIND=0.0.0.0"
   - "ZCASHD_RPCPORT=8232"  # Mainnet
   # - "ZCASHD_RPCPORT=18232"  # Testnet
   - "ZCASHD_ALLOWIP=0.0.0.0/0"  # Allow from anywhere (use with caution)
   ```

-> Раскомментируйте RPC-порт в *expose*:

   **Для Mainnet:**

   ```yaml
   - port: 8232
     as: 8232
     to:
       - global: false  # Keep internal for security
     proto: tcp
   ```

   **Для Testnet:**

   ```yaml
   - port: 18232
     as: 18232
     to:
       - global: false
     proto: tcp
   ```

**Предупреждение**: если вы установите *global: true* для RPC, вы откроете его в интернет с basic auth. Это плохая идея. Используйте *global: false* и получайте доступ к RPC через внутреннюю сеть Akash или настройте защищённый туннель.

**Напоминание о сопоставлении портов**: даже если вы откроете RPC глобально, Akash сопоставит его со случайным высоким портом (а не 8232/18232). Проверьте URIs в вашем развёртывании, чтобы увидеть фактический публичный эндпоинт. При *global: false* (рекомендуется) RPC-эндпоинт доступен только внутри сети развёртывания Akash, а не из публичного интернета.

### Включение индекса транзакций

Индекс транзакций позволяет запрашивать любую транзакцию по её ID через RPC. Использует больше хранилища (~ 20% прироста).

Раскомментируйте в *env*:

```yaml
- "ZCASHD_TXINDEX=1"
```

**Предупреждение**: включение txindex на уже синхронизированном узле требует повторной индексации всего блокчейна, что занимает часы.

### Включение Insight Explorer

Insight Explorer предоставляет дополнительные REST API-эндпоинты для данных блокчейна (полезно для обозревателей блоков).

Раскомментируйте в *env*:

```yaml
- "ZCASHD_INSIGHTEXPLORER=1"
```

Это автоматически включает txindex и добавляет дополнительные RPC-методы.

### Включение метрик Prometheus

Чтобы собирать метрики для мониторинга:

-> Раскомментируйте в *env*:

   ```bash
   - "ZCASHD_PROMETHEUSPORT=9969"
   - "ZCASHD_METRICSIP=0.0.0.0/0"
   ```

-> Раскомментируйте порт метрик в *expose*:

   ```bash
   - port: 9969
     as: 9969
     to:
       - global: false
     proto: tcp
   ```
   
Метрики будут доступны по адресу http://yourendpoint:9969/metrics в формате Prometheus.

### Настройка ресурсов/ценообразования

Если вы не получаете ставок или хотите оптимизировать стоимость:

**Для провайдеров с меньшими характеристиками** уменьшите значения в разделе *profiles.compute.zcashd.resources*:

-> CPU: *units: 2* (минимум для разумной скорости синхронизации)

-> Memory: *size: 12Gi* (минимум для стабильности)

-> Storage: *size: 120Gi* (минимум для mainnet)


**Чтобы привлечь больше ставок**, увеличьте значение в *profiles.placement.akash.pricing*:

-> Mainnet: попробуйте *amount: 15000* uakt/block

-> Testnet: попробуйте *amount: 7500* uakt/block


Значения в SDL установлены консервативно высокими. Большинство провайдеров предложат более низкую цену.

## Обновление вашего развёртывания

Нужно изменить конфигурацию после развёртывания?

-> Перейдите в **My Deployments** в Console

-> Найдите ваше развёртывание zcashd

-> Нажмите **"Update Deployment"**

-> Отредактируйте SDL

-> Нажмите **"Update"** и подтвердите в Keplr


**Примечание**: обновление перезапустит контейнер. Узел продолжит работу из сохранённого состояния (постоянное хранилище), но ожидайте 1-2 минуты простоя.

## Мониторинг

### Через Console

-> Вкладка **Logs**: журналы контейнера в реальном времени

-> Вкладка **Shell**: доступ к shell внутри контейнера (полезно для отладки)

-> Вкладка **Events**: события Kubernetes (в основном бесполезны, если ничего не сломалось)


### Через RPC (если включён)

Если вы включили RPC, вы можете запрашивать ваш узел как обычный полный узел zcashd (потому что это он и есть!)

### Альтернатива zcash-cli

Если у вас есть shell-доступ через Console, вы можете использовать *zcash-cli* напрямую:

```bash
# From the Shell tab in Console
zcash-cli getblockchaininfo
zcash-cli getpeerinfo
zcash-cli getinfo
```

## Закрытие вашего развёртывания

Когда закончите или захотите перестать платить:

-> Перейдите в **My Deployments**

-> Найдите ваше развёртывание zcashd

-> Нажмите **"Close Deployment"**

-> Подтвердите и подпишите в Keplr


Ваш депозит 5 AKT будет возвращён. **Постоянное хранилище** должно сохраняться провайдером, но не полагайтесь на это — относитесь к нему как к любому другому облачному провайдеру.

## Устранение неполадок

### Ошибка "Insufficient funds"

Вам нужно больше AKT. Пополните ваш кошелёк Keplr.

### Ставки не появляются

Либо:

-> Ваша цена слишком низкая (увеличьте *amount* в SDL)

-> Ваши требования к ресурсам слишком высоки для доступных провайдеров (уменьшите CPU/memory/storage)

-> Подождите дольше (иногда ставки появляются через 60-90 секунд)


### Развёртывание зависло в состоянии "pending"

У провайдера могут быть проблемы. Закройте развёртывание и попробуйте другого провайдера.

### В логах zcashd отображается "No peers connected"

Это нормально в первые несколько минут. zcashd автоматически обнаружит пиры. Если это продолжается более 10+ минут, у вас может быть проблема с сетью (в Akash это маловероятно).

### Ошибки "Out of memory" в логах

Вы слишком сэкономили на RAM. Закройте развёртывание и разверните заново как минимум с 12Gi памяти (рекомендуется 16Gi).

### Синхронизация занимает целую вечность

Уточним, что значит "целую вечность":

-> **Часы**: нормально

-> **Дни**: тоже нормально для mainnet с нуля

-> **Недели**: что-то не так, проверьте логи на ошибки


### "Error fetching zcash-params"

У провайдера могут быть сетевые проблемы или низкая пропускная способность. Обычно это решается само собой. Если проблема сохраняется более 30 минут, попробуйте заново развернуть у другого провайдера.

### Ошибки аутентификации RPC

-> Проверьте, что *ZCASHD_RPCUSER* и *ZCASHD_RPCPASSWORD* заданы правильно

-> Убедитесь, что используете правильный порт (8232 для mainnet, 18232 для testnet)

-> Помните, что порты сопоставляются Akash — используйте URI из вашего развёртывания, а не 8232 напрямую


## Управление стоимостью

Следите за расходами в Console:

-> **My Deployments** -> ваше развёртывание -> показывает оценку "Cost per month"

-> Баланс вашего кошелька Keplr со временем будет уменьшаться


Когда баланс станет низким, Akash автоматически закроет ваше развёртывание. **Периодически пополняйте кошелёк** или настройте оповещения.

### Снижение стоимости

-> **Используйте Testnet** для непродакшн-тестирования (на 50% дешевле)

-> **Уменьшите CPU/memory**, если вам не нужна быстрая синхронизация

-> **Выбирайте более дешёвых провайдеров** (не всегда разумно — аптайм важен)

-> **Используйте USDC вместо AKT**, если цена AKT волатильна (требуется изменить ценообразование в SDL)

-> **Отключите txindex**, если он вам не нужен (экономит ~ 20% хранилища)

## Заключительные замечания

- **Постоянное хранилище имеет значение.** Не пропускайте *persistent: true* и не используйте класс *beta2*. Используйте *beta3*.
- **Первичная синхронизация медленная.** Наберитесь терпения. Это нормально для узлов блокчейна.
- **Держите кошелёк пополненным.** Развёртывания автоматически закрываются, когда у вас заканчиваются AKT.
- **Резервные копии не создаются автоматически.** Если данные для вас важны, исходите из того, что они могут исчезнуть, и планируйте соответствующим образом.
- **Безопасность RPC критически важна.** Не открывайте RPC в интернет без надлежащих мер безопасности.
- **zcash-params кэшируются.** При первом запуске загружается ~2GB криптографических параметров. Это нормально и происходит только один раз.
