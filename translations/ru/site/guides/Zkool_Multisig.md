# Руководство по мультиподписи в Zkool

Это руководство содержит пошаговое описание того, как выполнять транзакции с мультиподписью с помощью Zkool. В него входят создание аккаунтов, отправка и получение средств, а также настройка распределённой генерации ключей (DKG) для мультиподписи. Для каждого основного шага приведены скриншоты.

## 1. Создание аккаунта


1. Откройте **приложение Zkool** и перейдите в **New Account**.


![img1](https://github.com/user-attachments/assets/ee906e49-361a-49b6-9484-904897fe2e3f)

3. Введите **название аккаунта** (например, Anabelle).  
   

![img2](https://github.com/user-attachments/assets/e9c325d3-8507-433a-a0c6-6e8c1ea2a254)


4. При необходимости включите **Use Internal Change** или **Restore Account**.


5. После создания аккаунт появится в вашем **списке аккаунтов**.  


![img3](https://github.com/user-attachments/assets/c446cbca-fb3e-49b9-b1d4-fd727cd1b0fb)


## 2. Получение средств

Каждый аккаунт генерирует несколько типов адресов:

**Unified Address**

**Адрес только Orchard**

**Адрес Sapling**
  
**Прозрачный адрес**


Выберите тип, который хотите использовать, и поделитесь им для получения средств.  


![img4](https://github.com/user-attachments/assets/c9de5dfe-e9d7-423d-8d90-35c1a08ffd5d)





## 3. Отправка средств

1. Перейдите в раздел **Recipient**.  


![img5](https://github.com/user-attachments/assets/9f3a03b9-dd56-450c-a8dc-4370f9289138)


3. Введите **адрес получателя**.  

4. Укажите **сумму** и при желании **memo**.  

5. Проверьте детали транзакции и **подтвердите**.  


После завершения баланс обновится в вашем списке аккаунтов.  


![img6](https://github.com/user-attachments/assets/6e6da76b-cd18-4567-a5c0-74f07ddefc64)


## 4. Выполнение транзакций с мультиподписью: настройка распределённой генерации ключей (Multisig)

Мультиподпись в Zkool использует **распределённую генерацию ключей (DKG)**, чтобы общий аккаунт контролировался несколькими участниками.



### Шаг 1: Инициация DKG
Выберите **название** для общего кошелька (например, Anabelle).

Укажите **количество участников**.
  
Выберите свой **ID участника**.
  
Определите **количество подписантов, необходимых для подтверждения (порог)**.
    
Выберите **аккаунт финансирования**.
  

![img7](https://github.com/user-attachments/assets/8a90ca85-5439-4937-b16d-a570e69d55f0)



### Шаг 2: Добавление адресов участников
- Введите **Unified Address** каждого участника (рекомендуется).


**Примечание:** Если вы используете адрес только Orchard или только Sapling, мультиподпись будет ограничена только этим пулом (Orchard или Sapling).  
Это означает, что общий кошелёк не сможет получать средства из других пулов.  
Для максимальной совместимости и гибкости всегда используйте **Unified Address**.  


### Шаг 3: Выполнение раундов DKG
Подождите, пока все участники обменяются пакетами **раунда 1** и **раунда 2**.  


![img8](https://github.com/user-attachments/assets/cdaf6e00-3cb0-4774-8a96-5ded19bf31c4)



### Шаг 4: Завершение формирования общего адреса
После завершения будет сгенерирован **общий адрес**.  


![img9](https://github.com/user-attachments/assets/741d1bc6-0102-4e67-bb83-9a1c184bd747)



## Заключение

Используя Zkool, вы можете: создавать аккаунты, отправлять и получать средства, а также настраивать **кошелёк с мультиподписью** с помощью Distributed Key Generation. Это обеспечивает **повышенную безопасность** и **совместное и приватное управление средствами**.
