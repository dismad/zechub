# **Arti: Tor-клиент нового поколения на Rust**
![Логотип Atri](https://gitlab.torproject.org/uploads/-/system/project/avatar/647/sliced-onions.png)

**Arti** — это инициатива Tor Project по созданию клиента **Tor** нового поколения с использованием языка программирования **Rust**. Arti спроектирован как модульный, встраиваемый и готовый к промышленному использованию клиент, обеспечивающий более безопасную и эффективную реализацию протоколов анонимности **Tor**. В **Arti версии 1.4.0** было представлено несколько значительных обновлений:

- **Новый интерфейс RPC** для расширенного взаимодействия.
- Подготовительная работа для **поддержки relay**.
- Улучшения в **устойчивости onion service на стороне сервиса к атакам типа "отказ в обслуживании"**.

Этот релиз продолжает усилия Tor Project по обеспечению лучшей безопасности, производительности и модульности для пользователей и разработчиков Tor.

## **Установка клиента Arti**

Следуйте этим шагам, чтобы установить и запустить **Arti** в качестве SOCKS-прокси в вашей системе.

---

### **Шаг 1: Настройте среду разработки Rust**

Прежде чем вы сможете собрать Arti из исходного кода, вам необходимо установить последнюю стабильную версию **Rust**.

#### Чтобы установить Rust:

1. Перейдите на официальный [сайт Rust](https://www.rust-lang.org/).
2. Следуйте инструкциям по установке для вашей операционной системы.
3. Проверьте установку, выполнив:
   
   ```sh
   rustc --version
   ```

Это подтвердит, что в вашей системе установлена последняя стабильная версия Rust.

#### **Примечание для пользователей Windows**:
- Rust можно установить в Windows через [**Rustup**](https://rustup.rs/) — инструмент для установки toolchain. Убедитесь, что вы также настроили совместимую среду сборки (в Windows вам могут понадобиться **Visual Studio Build Tools**).
  
---

### **Шаг 2: Клонируйте репозиторий Arti**

Чтобы получить последнюю версию клиента Arti, вам нужно клонировать репозиторий с [**GitLab**](https://gitlab.torproject.org/tpo/core/arti).

#### Шаги:
1. Откройте терминал (Command Prompt, PowerShell или Git Bash в Windows).
2. Выполните следующую команду, чтобы клонировать репозиторий:
   
   ```sh
   git clone https://gitlab.torproject.org/tpo/core/arti.git
   ```
4. Перейдите в только что созданный каталог *arti*:
   
   ```sh
   cd arti
   ```

Это загрузит исходный код Arti на ваш локальный компьютер.

---

### **Шаг 3: Соберите бинарный файл Arti**

После клонирования репозитория вам нужно собрать Arti с помощью **Cargo**, менеджера пакетов и инструмента сборки Rust.

#### Чтобы собрать Arti:
1. В терминале выполните следующую команду:
   ```sh
   cargo build --release
   ```

Эта команда компилирует код Arti и оптимизирует его для производственного использования (флаг *--release*). Бинарный файл будет создан в каталоге *target/release*.

#### Расположение скомпилированного бинарного файла:
- После сборки бинарный файл Arti будет находиться по адресу:  
  ```sh
  target/release/arti
  ```

Вы можете запускать этот бинарный файл напрямую из терминала.

---

### **Шаг 4: Запустите SOCKS-прокси Arti**

Чтобы использовать Arti как SOCKS-прокси (который будет направлять ваш интернет-трафик через сеть Tor), необходимо запустить прокси.

#### Чтобы запустить SOCKS-прокси:
1. Выполните следующую команду:
   ```sh
   ./target/release/arti proxy -p 9150
   ```

Эта команда запускает Arti как **SOCKS5-прокси** на **порту 9150**, который является портом по умолчанию, используемым Tor для SOCKS-трафика.

---

### **Шаг 5: Настройте приложения для использования Arti**

После того как Arti запущен как SOCKS-прокси, вам нужно настроить ваши приложения на его использование для маршрутизации трафика через сеть Tor.

#### Шаги:
1. В настройках вашего приложения (например, веб-браузера или терминального приложения) найдите **настройки прокси**.
2. Установите **SOCKS5-прокси** на *localhost:9150*.

Это направит весь трафик ваших приложений через **сеть Tor**, используя Arti в качестве промежуточного звена.

---

## **Интеграция Arti с сетью Tor**

Ниже приведена упрощённая схема, иллюстрирующая, как Arti работает совместно с сетью Tor:


```plaintext
[Application] --(SOCKS5)--> [Arti SOCKS Proxy] --(Tor Protocol)--> [Tor Network]
```

- **Приложение** подключается к **Arti SOCKS Proxy** по протоколу **SOCKS5**.
- Затем Arti взаимодействует с **сетью Tor**, обеспечивая анонимизацию вашего трафика при прохождении через сеть.

## **Репозиторий GitLab и участие в разработке**

Если вы заинтересованы в участии в разработке **Arti**, вы можете изучить код и внести свой вклад через **GitLab**.

- **Ссылка на репозиторий**: [Репозиторий Arti на GitLab](https://gitlab.torproject.org/tpo/core/arti)
- **Клонировать репозиторий**:
  ```sh
  git clone https://gitlab.torproject.org/tpo/core/arti.git
  ```

### **Форк и вклад в проект**:
1. **Сделайте fork** репозитория на GitLab (требуется учётная запись GitLab).
2. Свяжите ваш fork репозитория с локальной настройкой:
   ```sh
   git remote add _name_ git@gitlab.torproject.org:_name_/arti.git
   git fetch _name_
   ```
   Замените *_name_* на ваше имя пользователя GitLab.

3. **Отправьте изменения** в ваш fork:
   ```sh
   git push _name_ main
   ```

4. **Создайте Merge Request (MR)** на GitLab:
   Перейдите в раздел Merge Request в вашем GitLab fork:
   ```plaintext
   https://gitlab.torproject.org/_name_/arti/-/merge_requests
   ```

### **Рекомендации по Merge Request**:
- **Не выполняйте rebase и squash коммитов во время ревью**.
- При необходимости используйте *fixup!* или *squash!* для автоматического объединения коммитов.
- Старайтесь **добавлять новые коммиты**, а не объединять их в ходе цикла ревью.

---

### **Дополнительные примечания**:

- **Готовые бинарные файлы**: На данный момент **Arti** не предоставляет официальные готовые бинарные файлы. Вы должны собрать клиент из исходного кода, как описано выше.
- **Знание Rust**: Если вы участвуете в разработке Arti, имейте в виду, что кодовая база всё ещё развивается, и по мере добавления новых функций возможны изменения или рефакторинг.
