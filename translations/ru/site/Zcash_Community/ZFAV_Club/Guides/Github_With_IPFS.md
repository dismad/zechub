# Размещение репозитория Github через IPFS

## Введение

В этом руководстве мы узнаем, как создать URL, по которому можно клонировать git-репозиторий Github, размещённый с использованием IPFS CID. Это полезно для обеспечения доступности контента независимо от географического региона, устойчивости к цензуре и в качестве постоянной резервной копии ценной информации!

Примечание: Данные, загруженные в IPFS, доступны *всем* пользователям сети. Возможно, вы захотите локально зашифровать личные/конфиденциальные данные.


## Установка IPFS Kubo

Следуйте инструкциям по установке, приведённым [здесь](https://docs.ipfs.tech/install/command-line/#install-official-binary-distributions)

В этом примере мы используем Linux, также доступны версии для других ОС. 

Проверьте, что установка прошла успешно, с помощью команды "ipfs --version" 


## Клонирование репозитория

Для начала выберите Git-репозиторий, который хотите разместить, и клонируйте его:

Выполните команду: "git clone https://github.com/zechub/zechub"

![](https://i.ibb.co/HxFX37b/Screenshot-from-2023-05-20-14-14-46.png)


Теперь подготовим его к клонированию через IPFS.

cd zechub
git update-server-info


Распакуйте объекты Git:

mv objects/pack/*.pack .
git unpack-objects < *.pack
rm -f *.pack objects/pack/*

Это позволит IPFS устранять дублирование объектов, если позже вы обновите Git-репозиторий.


## Добавление в IPFS

После этого репозиторий готов к публикации. Остаётся только добавить его в IPFS:

$ pwd

/code/myrepo

$ ipfs add -r .

![](https://i.ibb.co/LJgK1q3/Screenshot-from-2023-05-20-14-22-38.png)

Полученный CID: Qmbgqox5g3614gjTb43s5mdSmmk95aGWWA9EHksL2T91A2

![](https://i.ibb.co/GvhCLwn/Screenshot-from-2023-05-20-14-26-34.png)

Отлично! Теперь ваш репозиторий загружен в сеть.


## Клонирование с использованием IPFS

Теперь вы должны иметь возможность получить github-репозиторий с помощью:

git clone http://ipfs.io/ipfs/"yourCID"

Кроме того, вы можете искать и получать его через ваш локальный узел IPFS. 

Заключительное примечание: Папка репозитория в IPFS не обновляется вместе с фактическим репозиторием github. Рекомендуется повторно загружать папку через регулярные промежутки времени.
