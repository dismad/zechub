# Опубликуйте веб-сайт в IPFS

![](https://blog.desdelinux.net/wp-content/uploads/2020/04/IPFS-.jpg)

## Введение в IPFS

IPFS (InterPlanetary File System) — это одноранговый протокол и сеть, предназначенные для создания децентрализованного способа хранения и обмена файлами.

В отличие от традиционной клиент-серверной модели интернета, IPFS позволяет пользователям обмениваться файлами напрямую друг с другом, а не полагаться на централизованный сервер для хранения и распространения контента.

Файлы в IPFS адресуются с использованием *content-addressing*, то есть каждому файлу присваивается уникальный хеш или CONTENT IDENTIFIER (CID) на основе его содержимого, и этот хеш используется для получения файла из сети.

Когда пользователь добавляет файл в IPFS, файл разбивается на небольшие части, называемые блоками, и каждому блоку присваивается CID. Затем эти блоки сохраняются на разных узлах сети, чтобы файл можно было легко получить из нескольких источников.

Это обеспечивает избыточность и отказоустойчивость, а также затрудняет превращение любого отдельного узла в единую точку отказа или контроля.

Прочитайте [Введение в IPFS](https://blog.infura.io/post/an-introduction-to-ipfs)



## Создание вашего сайта

В этом примере мы создаём простой веб-сайт.

[Пример сайта](https://squirrel.surf)


**Шаг 1:** Если вы не знакомы с веб-дизайном, напишите основной контент для вашего сайта, включая заголовок, основной текст, ссылки на другие страницы/сайты и футеры.

**Шаг 2:** Используйте [HTML-шаблон!](https://nicepage.com/html-templates) Вставьте написанный вами текст соответствующим образом. Также по желанию можно создать таблицу стилей .CSS для вашего сайта.

**Шаг 3:** Сохраните вашу директорию. Все страницы .html и изображения должны находиться в одной папке.



## Настройка узла

Скачайте и установите IPFS с [официального сайта](https://docs.ipfs.tech/install/ipfs-desktop/).



### Инициализация IPFS:

Если вы используете настольное приложение, инициализация не потребуется.

При использовании терминала или командной строки выполните команду: <mark>ipfs init </mark>.



**Добавьте папку сайта в IPFS**:

Выберите папку с файлами вашего сайта и перейдите к опции Add Folder.

![](https://i.ibb.co/ZHW4zsY/ipfs-site-folder.png)

--

Если вы используете терминал, выполните команду: <mark>ipfs add -r "folder_name"</mark>, чтобы рекурсивно добавить в IPFS всю папку.


### Закрепите сайт в IPFS:

После того как файлы вашего сайта добавлены в IPFS, вам нужно **закрепить** их, чтобы они оставались доступными в сети.

--

Если вы используете терминал, выполните команду: If using Terminal, Run command: <mark>ipfs pin add "hash"</mark>

"hash" = CID папки, которую вы добавили на предыдущем шаге.


Кроме того, вы также можете закреплять директории с помощью таких сервисов, как [Pinata](https://pinata.cloud) или [Dolpin](https://dolpin.io)

Это экономит много времени!

--

### Доступ к вашему сайту в IPFS:

Теперь ваш сайт опубликован в IPFS, и к нему можно получить доступ, используя хеш папки. Чтобы открыть ваш сайт, вы можете перейти по адресу https://ipfs.io/ipfs/"hash"

"hash" = CID папки.

В нашем случае CID = "QmW2UEfap1vrRRvS5H9wed8qmsx4WsvXBk3GPGVVfWx3r3"


## IPNS

Interplanetary Naming System (IPNS) позволяет вам обновлять CID IPFS, связанные с вашим сайтом, и при этом по-прежнему использовать статическую ссылку. Он предоставляется в виде ключа.

![](https://dnslink.io/assets/dns-query.a0134a75.png)

В меню настроек для папки вашего сайта в настольном приложении IPFS выберите Publish to IPNS.

![](https://i.ibb.co/Ch25dKf/IPNS.png)

Ключ: "k51qzi5uqu5di670a6uxywo17b2be1eyhoa2cl0qlwpfxn5p9ypcu8jbzgnj4n"

Его также можно использовать для просмотра нашего сайта через шлюз: https://ipfs.io/ipns/k51qzi5uqu5di670a6uxywo17b2be1eyhoa2cl0qlwpfxn5p9ypcu8jbzgnj4n


## DNS Link

Сайт создан, теперь нам нужен способ направить URL на этот контент.

Если у вас уже есть собственный веб-адрес, вы можете добавить новую запись, используя TXT-запись "_dnslink(your domain)". В зависимости от провайдера она может заполниться автоматически.

![](https://i.ibb.co/MgRxBHj/example.png)

Потребуется время, чтобы изменения распространились по сети, прежде чем вы сможете это увидеть.

Поздравляем! Вы настроили сайт, устойчивый к цензуре.
