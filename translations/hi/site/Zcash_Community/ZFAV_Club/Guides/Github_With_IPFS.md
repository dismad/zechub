# IPFS के साथ गिटहब रिपॉज़िटरी सर्व करें

## परिचय

इस मार्गदर्शक में हम सीखते हैं कि आप अपने गिटहब रिपॉज़िटरी के लिए एक गिट क्लोनेबल URL कैसे बनाते हैं, जो IPFS CID का उपयोग करके सर्व किया जाता है। इसके अतिरिक्त भूगोलिक क्षेत्र, प्रतिबंधन की रोकथाम और महत्वपूर्ण सूचना के लिए टिकाऊ बैकअप के रूप में यह उपयोगी है!

ध्यान दें: IPFS पर अपलोड की गई डेटा सभी नेटवर्क उपयोगकर्ताओं के लिए उपलब्ध है। आप स्थानीय रूप से व्यक्तिगत/संवेदनशील डेटा को एन्क्रिप्ट करना चाह सकते हैं।


## IPFS Kubo कोइन्स्टॉल करें

[यहाँ](https://docs.ipfs.tech/install/command-line/#install-official-binary-distributions) प्रदान की गई इस्ताल निर्देशों के साथ अनुसरण करें

इस उदाहरण में हम लिनक्स का उपयोग करते हैं, अन्य OS संस्करण उपलब्ध हैं। 

"ipfs --version" के उपयोग द्वारा प्रस्तुति की सफलता की जांच करें


## रिपॉज़िटरी क्लोन करें 

शुरू करने के लिए, अपने गिट रिपॉज़िटरी का चयन करें जिसे होस्ट करना आवश्यक है और इसे क्लोन करें:

आदेश चलाएं: "git clone https://github.com/zechub/zechub"

![](https://i.ibb.co/HxFX37b/Screenshot-from-2023-05-20-14-14-46.png)


अब, इसे IPFS के माध्यम से क्लोन करने के लिए तैयार करें।

cd zechub
git update-server-info


गिट के ऑब्जेक्ट्स को अपकैक करें:

mv objects/pack/*.pack .
git unpack-objects < *.pack
rm -f *.pack objects/pack/*

यह आपके गिट रिपॉज़िटरी के बाद में अपडेट करने की अनुमति देता है।


## IPFS पर जोड़ें 

एक बार इसे करने के बाद, वह रिपॉज़िटरी सर्व करने के लिए तैयार है। अब केवल IPFS में इसे जोड़ना बचा है:

$ pwd

/code/myrepo

$ ipfs add -r .

![](https://i.ibb.co/LJgK1q3/Screenshot-from-2023-05-20-14-22-38.png)

परिणामी CID: Qmbgqox5g3614gjTb43s5mdSmmk95aGWWA9EHksL2T91A2

![](https://i.ibb.co/GvhCLwn/Screenshot-from-2023-05-20-14-26-34.png)

ब्रिलियंट! अब आपका रिपॉज़िटरी नेटवर्क पर अपलोड कर दिया गया है।


## IPFS के साथ क्लोन करें 

अब आपको इसके माध्यम से गिटहब रिपॉज़िटरी प्राप्त करना चाहिए:

git clone http://ipfs.io/ipfs/"yourCID"

एक वैकल्पिक रूप से, आप अपने स्थानीय IPFS नोड के माध्यम से खोज और प्राप्त कर सकते हैं। 

अंतिम टिप्पणी: IPFS पर रिपॉज़िटरी फ़ोल्डर में वास्तविक गिटहब रिपॉज़िटरी के साथ अपडेट प्राप्त नहीं होता। यह अनुशंसा की जाती है कि आप नियमित अंतराल पर फ़ोल्डर को फिर से अपलोड करें।
