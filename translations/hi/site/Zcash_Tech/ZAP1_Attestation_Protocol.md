# ZAP1 प्रमाणन प्रोटोकॉल

ZAP1 एक ओपन सोर्स प्रमाणन प्रोटोकॉल है जो Zcash के लिए डिज़ाइन किया गया है। यह संरचित जीवन चक्र घटनाओं को BLAKE2b मेर्कल ट्री में लिखता है और ट्री के मूल को Orchard छिपे हुए स्मैक्स के माध्यम से चेन पर एनकर करता है। प्रूफ सार्वजनिक रूप से सत्यापित करे जा सकते हैं। घटना डेटा निजी रहता है।

## कैसे काम करता है

ऑपरेटर्स घटना प्रकार (जैसे, तैयारी, भुगतान, स्थानांतरण) को पंजीकृत करते हैं और उन्हें ZAP1 के एक इंस्टेंस में भेजते हैं। प्रत्येक घटना डोमेन-विभाजित BLAKE2b-256 के उपयोग से एक पत्ती हैश बनाती है। पत्तियां मेर्कल ट्री में जमा होती हैं। जब एक थ्रेशहोल्ड प्राप्त कर लिया जाता है, तो ट्री के मूल को ZAP1:09 स्मैक्स के रूप में कोडित करते हुए Zcash में एनकर कर दिया जाता है।

कोई भी पत्ती हैश के साथ लीफ से मूल और चेन पर एनकर तक के पूरे पथ की सत्यापन कर सकता है, ऑपरेटर पर विश्वास किए बिना।

## मुख्य गुण

- **एप्लिकेशन अग्नोस्टिक**: कोई भी Zcash ऑपरेटर अपने स्वयं के घटना प्रकार और व्यक्तिगतकरण स्ट्रिंग्स को परिभाषित कर सकता है
- **गोपनीयता बचाने वाला**: घटना भार एनकर करने से पहले हैश कर दिया जाता है। केवल हैश चेन पर जाता है।
- **स्वतंत्र रूप से सत्यापन योग्य**: सत्यापन में केवल प्रूफ बॉडले और चेन एक्सेस की आवश्यकता होती है। ऑपरेटर पर विश्वास की आवश्यकता नहीं होती है।
- **ZIP 302 संगत**: ZAP1 ZIP 302 के लिए attestation payload के लिए partType की ओर बढ़ रहा है

## क्या मौजूद है

- संदर्भ प्रायोग (Rust, MIT लाइसेंस)
- crates.io पर सत्यापन SDK (Rust + 83KB WASM)
- npm पर JavaScript SDK
- सार्वभौमिक स्मैक्स डीकोडर (ZAP1, ZIP 302 TVLV, टेक्स्ट, बाइनरी और खाली स्मैक्स की पहचान करता है)
- 29 API जांचों और 14 प्रोटोकॉल जांचों के साथ सम्मिलन किट
- मल्टीपार्टी एनकर ब्रॉडकास्टिंग के लिए FROST 2-of-3 थ्रेशहोल्ड साइनिंग डिज़ाइन
- ZIP प्रस्ताव परीक्षण #1243 में समीक्षा के अंतर्गत
- मार्च 2026 तक 4 मुख्य नेट एनकर और 14 पत्तियों

## आर्किटेक्चर

```
आपका एप्लिकेशन --> ZAP1 API --> Merkle tree --> Zcash anchor
                  |                                    |
             event types                         shielded memo
          (DEPLOYMENT, etc)                    (ZAP1:09:{root})
```

प्रत्येक ऑपरेटर अपने स्वयं के कुंजी, Merkle tree और anchors के साथ अपना ZAP1 इंस्टेंस चलाता है। ऑपरेटर्स के बीच कोई साझा स्टेट नहीं होता है।
