# Brave Wallet के साथ Shielded Zcash (ZEC) कैसे भेजें और प्राप्त करें: चरण-दर-चरण गोपनीयता मार्गदर्शिका

Brave Wallet एक सुरक्षित, native crypto wallet है जो सीधे Brave browser में बनाया गया है - किसी extension की आवश्यकता नहीं।


## Shielded ZEC सपोर्ट क्यों महत्वपूर्ण है


Zcash उन कुछ cryptocurrencies में से एक है जो shielded transactions के साथ डिफ़ॉल्ट रूप से गोपनीयता प्रदान करती हैं।

Brave Wallet अब Orchard shielded ZEC को सपोर्ट करता है, जिससे third-party software के बिना private transactions संभव हो जाती हैं।

अब उपयोगकर्ता balances या metadata प्रकट किए बिना ZEC भेज और प्राप्त कर सकते हैं - वह भी पूरी तरह Brave browser के भीतर।

## भाग 1: Brave Wallet सेट अप करना

**शामिल चरण:**

1. Brave Browser खोलें (फिलहाल केवल desktop के लिए)

2. Wallet आइकन पर क्लिक करें या brave://wallet पर जाएँ

3. अपना wallet सेट अप करें (बनाएँ या import करें)

4. crypto dashboard तक पहुँचें

## भाग 2: Brave Wallet में Zcash (ZEC) जोड़ना

**शामिल चरण:**

1. Wallet dashboard में "Manage Assets" पर क्लिक करें

2. ZEC खोजें और उसे enable करें

3. अब ZEC आपके wallet में दिखाई देगा


![img2](https://github.com/user-attachments/assets/6f2b2190-cf55-4394-9d5f-29ff9b5bb525)


## भाग 3: Shielded ZEC प्राप्त करना (Unified Address)

**शामिल चरण:**

1. अपनी assets में ZEC पर क्लिक करें

2. "Receive" चुनें

3. Brave एक Unified Address (UA) बनाता है - एक ऐसा address जो shielded और transparent दोनों प्रकार के transactions को सपोर्ट करता है

4. QR code को कॉपी करें या scan करें

5. अपना UA प्रेषक के साथ साझा करें



![img3](https://github.com/user-attachments/assets/53c940b6-1a03-4fa7-aefa-d3478f678a88)



## भाग 4: निजी रूप से Shielded ZEC भेजना

**शामिल चरण:**

1. ZEC -> Send चुनें

2. प्राप्तकर्ता का Unified Address या shielded address पेस्ट करें

3. राशि दर्ज करें

4. transaction विवरण की पुष्टि करें

5. Send पर क्लिक करें - अब धनराशि Orchard pool का उपयोग करते हुए निजी रूप से ट्रांसफ़र हो जाती है

## Address के प्रकार

यहाँ Zcash के प्रत्येक address प्रकार का विवरण दिया गया है:

**t-addr (t1...)** - Transparent addresses Bitcoin addresses की तरह काम करते हैं, यानी transactions blockchain पर सार्वजनिक होती हैं। इन्हें non-Zcash wallets के साथ उपयोग करना आसान होता है, लेकिन ये गोपनीयता प्रदान नहीं करते।

**z-addr (zs...)** - Shielded addresses, Zcash की privacy tech (Sprout/Sapling) का उपयोग करके sender, receiver और amount को छिपाते हैं। केवल view keys या मालिक ही विवरण प्रकट कर सकते हैं।

**UA (u... or QR)** - Unified Address कई address types (transparent, shielded, Orchard) को एक में जोड़ते हैं। इससे अधिकतम compatibility मिलती है, जबकि जहाँ संभव हो वहाँ गोपनीयता को प्राथमिकता दी जाती है।
