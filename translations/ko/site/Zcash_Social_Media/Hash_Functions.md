## 1부: 해시 함수

오늘 우리는 **해시 함수**에서 시작합니다. 블록체인에 사용되는 암호학의 핵심 요소입니다. 이 시리즈 후반에는 이러한 속성에 의존하는 주제들에 대해 다루게 됩니다.

### 해시 함수란 무엇인가?

해시 함수는 입력 길이와 관계없이 고정된 길이의 출력을 생성합니다.

- **해싱할 메시지** = 입력  
- **사용되는 알고리즘** = 해시 함수  
- **결과 출력** = 해시 값

### 암호학적 해시 함수의 속성

암호학적 해시 함수는 다음 **3가지 속성을** 가져야 합니다:

1. **일방향성** - 해시 함수를 역추적하는 것은 불가능해야 함  
2. **충돌 방지** - 두 개의 다른 입력이 동일한 출력으로 해싱되어서는 안 됨  
3. **결정론적** - 어떤 입력에 대해서도 해시 함수는 항상 동일한 결과를 제공해야 함

### 일반적인 해시 함수

해시 함수에는 여러 가지 유형이 있습니다. 몇 가지 예시:

- 보안 해싱 알고리즘 (**SHA-3**)  
- 메시지 다이제스트 알고리즘 5 (**MD5**)  
- **BLAKE2b** - Zcash 키 도출에 사용됨

**Zooko의 BLAKE2 소개**: https://www.zfnd.org/blog/blake2/

### 해시 함수의 실제 활용 사례

#### 1. 무결성 해싱 (데이터 무결성 검사)
데이터 무결성 검사는 "무결성 해싱"의 예입니다. 데이터 파일에 체크섬을 생성하고 사용자에게 정확성을 보장하는 데 사용됩니다.

![무결성 해싱 예시](https://pbs.twimg.com/media/Fn_Or0MWIAI6sgx.png)

#### 2. 머클 트리 (해시 트리)
**해시 트리**(또는 **머클 트리**)는 데이터 블록의 암호학적 해시로 라벨이 붙은 가지와 리프 노드로 구성됩니다.

![머클 트리 다이어그램](https://pbs.twimg.com/media/Fn_O7ndWIAY5PA-.jpg)

머클 트리는 **암호학적 약속 체계**(cryptographic commitment scheme)의 예입니다. 트리 루트는 약속으로 간주되고, 리프 노드는 원래 약속에 포함되어 있음을 증명합니다.

P2P 네트워크에서 저장되거나 전송되는 데이터를 검증하는 데 사용됩니다. 피어로부터 받은 데이터가 변경되지 않았음을 보장합니다.

#### 3. Zcash의 노트 커밋먼트 트리
Zcash **Sapling** 및 **Orchard** 가상 지갑에서 **노트 커밋먼트 트리**(Note Commitment Tree)는 합의에 따라 거래가 유효함을 검증하면서 송신자, 수신자 및 소비된 금액을 완벽하게 숨기는 데 사용됩니다.

#### 4. 서명 해시 (비트코인 스타일 블록)
**SHA256**은 "서명 해시"의 예로, 비트코인 체인에서 각 블록의 불변성을 보장하는 데 사용됩니다. 마이너는 이전 블록의 해시 + 현재 블록 내 모든 거래의 해시(해시 머클 루트) + 타임스탬프 + 무작위 값 / 네트워크 난이도를 새로운 블록에 사용합니다.

![SHA256 블록 다이어그램](https://pbs.twimg.com/media/Fn_PaVZXoAApHPf.jpg)

#### 5. Equihash (Zcash 채굴)
**Equihash**는 Zcash를 채굴하는 데 사용되는 해시 알고리즘입니다. Komodo 및 Horizen 네트워크에서도 사용됩니다.

**원래 Zcash의 Equihash 블로그**: https://electriccoin.co/blog/equihash/
