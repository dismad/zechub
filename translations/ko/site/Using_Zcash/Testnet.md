# Zcash 테스트넷

## Zcash 테스트넷이란?

**Zcash 테스트넷**(Testnet)은 실제 Zcash 메인 네트워크(Mainnet)와 병렬적으로 작동하는 블록체인으로, 정확한 프로토콜, 규칙 및 거래 논리를 복제하지만 두 가지 주요 차이점이 있습니다:

1. **코인에는 실제 통화 가치가 없습니다** - 이는 **TAZ**(Testnet Zcash)라고 불리며, 테스트 용도로만 사용됩니다.
2. **네트워크 업그레이드, 도구 및 소프트웨어는 먼저 여기서 테스트되고 배포됩니다**.

즉, 테스트넷은 개발자, 감사자 및 구축자가 실제 자금을 위험에 빠뜨리지 않고 아이디어를 시도할 수 있는 **샌드박스 또는 실험 환경**과 같습니다.


## 왜 테스트넷이 존재하는가?

테스트넷은 블록체인 개발에 필수적이며, **실제 블록체인(Zcash)은 불변**입니다. 메인 네트워크에서 거래가 확인되면 되돌릴 수 없습니다. 테스트넷은 실제 Zcash 블록체인에 배포하기 전에 기능을 실험, 테스트 및 디버깅할 수 있는 **안전한 복제본**을 제공합니다.

### 테스트넷의 용도

#### 1. 소프트웨어 개발 및 통합

지갑, 거래소, 채굴 소프트웨어 또는 프라이버시 도구를 개발하는 개발자는 테스트넷에서 안전하게 이를 테스트할 수 있습니다. 제공되는 기능은 다음과 같습니다:

- 거래 송수신  
- 가치 없는 TAZ 코인으로 새로운 블록 채굴  
- 사용자 인터페이스 및 API 구축  
- 거래 프라이버시 기능(투명 vs 보호) 테스트  

**예시:**  
[`zcash_tx_tool`](https://github.com/QED-it/zcash_tx_tool) 같은 도구는 테스트넷을 통해 거래를 생성하고 Zcash 보호 자산 기능성을 테스트합니다.

**실제 시나리오:**  
지갑 개발자는 메인 네트워크에 연결하기 전에 테스트넷 RPC 엔드포인트에 소프트웨어를 연결하여 전체 생명주기를 시뮬레이션할 수 있습니다. 주소 생성, 보호 거래 송신 및 잔액 검증을 포함합니다.

#### 2. 네트워크 업그레이드 테스트

Zcash는 주기적으로 핵심 프로토콜을 업데이트(예: NU5, NU6). 테스트넷은 메인넷보다 먼저 새로운 업그레이드를 활성화하여 개발자와 커뮤니티가 버그를 식별하고 수정할 수 있도록 합니다.

**예시:**  
새로운 합의 규칙 또는 거래 유형이 처음으로 테스트넷에 푸시됩니다. 성공적인 테스트 후, 메인넷에서 특정 블록 높이에서 활성화됩니다.

#### 3. 노드 구현 테스트

Zcash는 여러 노드 소프트웨어 구현을 지원합니다 - `zcashd` 및 **Zebra**(Zcash Foundation이 유지하는 Rust 기반 노드). 테스트넷은 실제 조건에서 노드를 테스트할 수 있도록 하여 재정적 위험 없이 가능하게 합니다.

노드 개발자는 다음과 같은 작업을 수행할 수 있습니다:

- 블록 전파 검증  
- RPC 인터페이스 테스트  
- 부하 상태에서 노드 행동 관찰  
- 채굴 소프트웨어 상호작용 테스트  

#### 4. 학습 및 교육

초보자는 채굴, 보호 거래 생성, 통합 주소 사용과 같은 Zcash 기능을 배울 수 있습니다.
커뮤니티 튜토리얼 및 문서는 **테스트넷 faucets, 탐색기 및 가이드**에 대한 접근성을 제공합니다.

## 실제 테스트넷 사용 사례

### 1. 개발자 테스트(지갑 / 앱)

- Zcash 테스트넷 연결  
- faucets에서 TAZ 요청  
- 보호 거래 송신  
- 프라이버시 및 UI 안정성 검증  

실제 ZEC가 실수로 잃어도 되지 않습니다.

### 2. 거래소 통합 테스트

- 테스트넷 노드 실행  
- Zebrad JSON-RPC 엔드포인트를 사용하여 거래 처리  
- 자동 입금/출금 로직 테스트  

생산 코드의 안전성 보장 및 재정적 손실 방지.

### 3. 채굴 설정 시험

- 채굴 템플릿 사용  
- 블록 검증 테스트  
- 채굴 보상(TAZ만) 관찰  
- 채굴 성능 조정  

메인넷으로 이전할 때 다운타임 또는 수익 손실 방지.

### 4. 학술 / 프로토콜 연구

연구자들은 **무상 검증**, **제로 지식 증명 최적화** 또는 다른 프로토콜 실험을 테스트넷에서 테스트할 수 있습니다.
고급 사용자는 또한 **커스텀 테스트넷 또는 regtest 환경**을 실행하여 전문적인 실험을 수행할 수 있습니다.

## 메인넷과 테스트넷의 주요 차이점

| 기능               | 메인넷           | 테스트넷                  |
|-------------------|-----------------|--------------------------|
| 코인 가치         | 실제 ZEC        | TAZ (통화 가치 없음)     |
| 위험              | 재정적 위험      | 테스트용 안전            |
| 프로토콜 업그레이드 | 운영 중          | 조기 활성화             |
| 채굴 보상         | 실제 발행        | 테스트 보상만           |
| 네트워크 유틸리티  | 실거래           | 테스트 및 개발           |

## 일반적인 오해

- **테스트넷 코인은 가치가 있다** -> 잘못된 정보, TAZ는 가치가 없습니다.
- **테스트넷 코인을 잃는 것은 중요하다** -> 잘못된 정보, 실제 가치 손실이 없습니다.
- **테스트넷과 메인넷은 동일하다** -> 잘못된 정보, 테스트넷은 자주 리셋되고 메인넷만큼 경제적으로 보호되지 않습니다.

---

## TAZ란?

**TAZ**(Testnet Zcash)는 Zcash 코인의 테스트넷 버전입니다:  

- 실제 통화가 아니며, ZEC 또는 현금으로 교환할 수 없음  
- 테스트, 개발 및 학습용  
- 모든 Zcash 규칙을 따름: 송신, 채굴, 보호 주소 사용 가능  

**예시:**  
개발자는 실제 ZEC를 위험에 빠뜨리지 않고 테스트넷 주소 간 100 TAZ 송신으로 지갑 기능을 테스트할 수 있습니다.  

TAZ는 **"Zcash 테스트넷용 플레이 머니"**로 생각해보세요.


## faucets란?

**faucet**( faucets)는 테스트 용도로 무료 TAZ 코인을 제공하는 서비스입니다:

- 일반적으로 웹사이트 또는 API  
- 사용자가 테스트넷 주소를 제공하면, faucet이 작은 양의 TAZ를 보냅니다.  
- TAZ를 수동으로 채굴할 필요가 없습니다.

**예시:**  
1. [testnet.zecfaucet.com](https://testnet.zecfaucet.com) 또는 [fauzec.com](https://fauzec.com/) 같은 테스트넷 faucets에 방문합니다.
2. 테스트넷 주소를 입력합니다.
3. TAZ 요청을 합니다.
4. 즉시 TAZ를 받고 테스트를 시작합니다.

**중요성:**  
- 실제 ZEC 위험 없이 안전한 테스트 가능  
- 초보자 및 개발자에게 접근성 제공  
- 지갑, 거래소, 앱의 빠른 프로토타입 설계

## Zkool 및 Zingo! 지갑

### Zkool

- 고급 Zcash 사용자를 위한 다중 계정 지갑
- 시드 구문, 보기 키, 투명하고 보호 주소 지원
- 전체 노드 또는 라이트월릿 서버를 통해 메인넷, 테스트넷 또는 regtest에 연결 가능

### Zingo!

- 프라이버시와 간단함을 중점으로 한 모바일 지갑
- 보호 및 통합 주소 지원
- NU6 테스트넷 포함한 테스트넷 프로토콜 지원 업데이트됨

## 테스트넷을 활성화한 후

- 거래는 메인넷과 동일하지만 **가치 없는 TAZ**로 이루어짐  
- 보호 거래, 여러 주소 및 프라이버시 기능이 테스트 가능  
- 개발자는 실제 ZEC 위험 없이 기능을 디버깅하고 테스트할 수 있음

## 간단한 요약

- **Zcash 테스트넷**은 빌드, 테스트 및 실험에 안전한 샌드박스 환경입니다  
- 사용 사례: 개발자 테스트, 노드 테스트, 거래소 통합, 연구 및 교육  
- **TAZ 코인**은 ZEC 대신 사용되며 실제 가치가 없음  
- 메인넷에 기능을 라이브로 배포하기 전에 테스트넷이 필수적입니다
