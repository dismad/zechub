# Zcash의 TEX 주소란 무엇인가요?

Zcash의 TEX 주소는 독특한 유형의 수신 주소를 나타냅니다. "Transparent Exchange"를 의미하는 약어로, 단일 p2pkh 투명 주소의 **유일성**과 통합 형식(bech32m)으로 인코딩된 주소입니다.

이 주소의 목적은 호환되는 지갑에 T -> T(투명 전용) 거래를 수행하도록 알리는 것입니다.

논리적 흐름은 다음과 같습니다: 호환되는 지갑이 TEX 주소를 감지하면 이를 해독하여 내부에 포함된 투명 수신자 주소를 얻습니다. 이후 지갑은 tx에 필요한 자금을 Shielded 풀에서 사용자가 제어하는 일시적인 투명 주소(Z -> T)로 이동시키고, 그 후 해당 자금을 TEX 주소의 해독된 투명 수신자(T -> T)로 전송합니다.

TEX 주소에 대한 기술적 제안은 Zcash [ZIP 320](https://zips.z.cash/zip-0320)에서 정의되어 있으며, 이는 투명 주소로부터 자금을 수신하기 위해 독점적으로 사용되는 주소 유형을 설명합니다.

![TEX](https://i.ibb.co/8m7HPqV/ZashiTex.png)

TEX 주소는 아직 널리 채택되지 않았지만, Zcash 사용자는 결국 이를 사용해야 할 수도 있습니다.

## 언제 TEX 주소가 필요한가요?

### 투명 주소로 자금을 보내려고 하는데, 해당 지갑이 투명 주소로 직접 보낼 수 없는 경우 **TEX 주소가 필요**합니다.
일부 지갑은 투명 주소로 직접 보내는 것을 허용하지 않으며, **수신자가 TEX 대응 주소를 제공하지 않을 수도 있습니다.** 따라서, 때때로 투명 주소에서 TEX 주소로의 **변환**이 필요할 수 있습니다. 이는 zip-320에 명시된 참조 구현을 실행하여 수동으로 수행할 수 있습니다. **투명 주소에서 TEX 주소로 변환하는 호스팅 인스턴스**는 [여기](https://690e9524c66a3ecac5d54eff--jade-brioche-873777.netlify.app/)에서 확인할 수 있습니다.

### 중앙 집중형 거래소에 자금을 보내려고 하는데, **투명 출처에서의 자금이 필요하다면** TEX 주소가 필요합니다.
현재 [Binance](https://www.binance.com/)는 TEX 주소를 사용하는 유일한 중앙 집중형 거래소이며, 이들이 TEX 주소 생성의 주요 이유입니다.
TEX 주소는 호환되는 지갑에 해당 주소로 보내는 자금이 모두 투명해야 하며, 모든 Shielded 값이 해당 주소로 전송되지 않도록 알려줍니다.
Binance와 같은 거래소가 보낸 금액을 거부할 경우, 그 금액을 원래 출처 주소로 되돌려보낼 수 있는 방법을 갖게 됩니다. 또한 Binance와 같은 업체는 정부나 다른 당국이 요구하는 법률 및 규정 준수에 도움을 줍니다.

## 어떤 지갑들이 TEX 주소를 지원하나요?

최신 정보는 우리의 [지갑](https://zechub.wiki/wallets) 페이지에서 확인할 수 있습니다. **TEX 주소 필터**를 사용하세요.
