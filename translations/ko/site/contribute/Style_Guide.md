# ZecHub 스타일 가이드

ZecHub의 스타일은 간단하고 접근성이 높습니다. 누구나 환영받고, Zcash 커뮤니티에 초점을 맞추고 있습니다.

## 원칙

1. **도움을 제공 준비가 되어 있음** - Zcash 사용자들이 우리가 그들과 함께 있다는 것을 보여줍니다.

2. **간결하고 핵심에 집중:** 가장 간단한 메시지를 전달하세요. 중요한 내용부터 시작하세요.

3. **사용자 중심:** 사용자를 우선으로 하되, 개발자는 두 번째로, 나머지는 세 번째로 고려합니다.

## 기본 원칙

- **핵심에 직설적으로**. 가장 중요한 점부터 시작하세요. 키워드를 앞쪽에 배치하여 빠르게 스크롤할 수 있도록 하세요. 사용자의 선택지와 다음 단계가 명확하게 되도록 하세요.

- **말하는 것처럼 쓰세요**. 우리의 사용자는 실제 사람입니다. 글로 그들과 대화하세요.

- **간결하게**. 사용자가 문제를 해결하거나 결정을 내릴 수 있도록 충분한 정보를 제공하세요.

- **제목에 마침표 생략**. 제목, 헤딩, 서브헤드, UI 제목 및 3개 이하의 단어로 된 목록 항목에서 끝부분의 구두점을 제거하세요. 마침표는 단락과 본문에 사용합니다.

  - 예시: Press the Send button → Press the Send button

- **불확실할 때는 대문자 생략**. Zcash와 같은 고유명사나 헤딩 또는 문장의 첫 번째 단어만 대문자를 씁니다.

  - 예시: How to Send ZEC with Unified Address → How to send ZEC with unified address

- **약한 표현을 수정하세요**. 대부분의 경우, 각 문장을 동사로 시작하세요. "You can" 또는 "there were" 같은 표현은 수정해야 합니다.

  - 예시: You can send and receive ZEC from your mobile phone → Send and receive ZEC with your phone

- **공백을 적절히 사용**. 마침표와 콜론 뒤에는 하나의 공백만 사용하세요.

- **비영어 단어를 피하세요**, 예를 들어 de facto 또는 ad hoc 같은 표현. 일반적인 영어 문장의 라틴어 약자도 피하세요.

    | 사용할 것       | 대신 사용할 것     |
    |----------------|--------------------|
    | for example    | e.g.               |
    | that is        | i.e.               |
