# Brave Wallet으로 Shielded Zcash (ZEC) 보내고 받기: 단계별 프라이버시 가이드

Brave Wallet은 Brave 브라우저에 직접 내장된 안전한 네이티브 크립토 지갑으로, 확장 프로그램이 필요 없습니다.


## Shielded ZEC 지원이 중요한 이유


Zcash는 shielded 트랜잭션을 통해 기본적으로 프라이버시를 제공하는 몇 안 되는 암호화폐 중 하나입니다.

이제 Brave Wallet은 Orchard shielded ZEC를 지원하여, 서드파티 소프트웨어 없이도 비공개 트랜잭션을 가능하게 합니다.

이제 사용자는 Brave 브라우저 안에서 잔액이나 메타데이터를 노출하지 않고 ZEC를 송금/수신할 수 있습니다.

## 1부: Brave Wallet 설정하기

**포함된 단계:**

1. Brave 브라우저 열기(현재는 데스크톱만 지원)

2. Wallet 아이콘을 클릭하거나 `brave://wallet`으로 이동

3. 지갑 설정하기(새로 생성 또는 가져오기)

4. 크립토 대시보드에 접속하기


![img1](https://github.com/user-attachments/assets/f54cd1a1-8569-4925-ba1c-7597d030593e)


## 2부: Brave Wallet에 Zcash (ZEC) 추가하기

**포함된 단계:**

1. Wallet 대시보드에서 "Manage Assets"를 클릭

2. ZEC를 검색해 활성화

3. 이제 지갑에 ZEC가 표시됨


![img2](https://github.com/user-attachments/assets/6f2b2190-cf55-4394-9d5f-29ff9b5bb525)


## 3부: Shielded ZEC 받기 (Unified Address)

**포함된 단계:**

1. 자산 목록에서 ZEC를 클릭

2. "Receive" 선택

3. Brave가 Unified Address (UA)를 생성함 - shielded 및 transparent 트랜잭션을 모두 지원하는 하나의 주소

4. QR 코드를 복사하거나 스캔

5. 발신자와 UA를 공유



![img3](https://github.com/user-attachments/assets/53c940b6-1a03-4fa7-aefa-d3478f678a88)



## 4부: Shielded ZEC를 비공개로 보내기

**포함된 단계:**

1. ZEC -> Send 선택

2. 수신자의 Unified Address 또는 shielded 주소 붙여넣기

3. 금액 입력

4. 트랜잭션 세부 정보 확인

5. Send를 클릭 - 이제 자금이 Orchard 풀을 사용해 비공개로 전송됩니다


![img4](https://raw.githubusercontent.com/Kellyjoe8/zechub/refs/heads/main/Internet_20250808_172118_4.webp)




## 주소 유형

다음은 각 Zcash 주소 유형에 대한 설명입니다:

**t-addr (t1...)** - Transparent 주소는 Bitcoin 주소처럼 작동하므로, 트랜잭션이 블록체인에 공개됩니다. 비Zcash 지갑과 함께 사용하기는 더 쉽지만 프라이버시를 제공하지는 않습니다.

**z-addr (zs...)** - Shielded 주소는 Zcash의 프라이버시 기술(Sprout/Sapling)을 사용해 발신자, 수신자, 금액을 숨깁니다. Viewing Key 또는 소유자만 세부 정보를 확인할 수 있습니다.

**UA (u... or QR)** - Unified Address는 여러 주소 유형(transparent, shielded, Orchard)을 하나로 결합합니다. 이를 통해 가능한 경우 프라이버시를 우선하면서도 최대한의 호환성을 제공합니다.
