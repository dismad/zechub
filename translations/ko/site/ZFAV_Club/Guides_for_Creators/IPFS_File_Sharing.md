# IPFS와 브레이브 브라우저를 이용한 P2P 파일 공유

우리는 대규모 미디어 파일을 서로 간에 Peer to-Peer 방식으로 공유하기 위해 IPFS를 사용할 수 있습니다.

문제는 무엇인가요? IPFS 게이트웨이는 대규모 파일에는 작동하지 않으며, 시간 초과가 발생합니다.

**필수 조건**: [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) + [Brave Browser](https://brave.com/)

tldr, 동영상은 마지막에 있습니다.

ZF AV 클럽에서 서로 간에 대규모 미디어 파일을 전송하고, 여러분 모두에게 이를 기록할 수 있습니다. 이는 재미있고 매우 멋진 일입니다!!

*PLUS*, 우리는 Protocol Labs의 IPFS 팀과 직접 협력하여 이 사용 사례를 개발할 것입니다.
이것은 dWeb 캠프에서 맺은 관계로부터 얻은 또 다른 이점입니다!

수신 기기에 IPFS가 설치되지 않았거나, 브레이브가 아닌 브라우저를 사용하는 경우: 파일을 로드하기 위해 게이트웨이를 기다려야 합니다. *대규모 파일의 경우 504 오류가 발생합니다.*

다음 단계: 다른 사람들의 파일을 자신의 기기에서 고정(pin)하여, 원본 노드가 오프라인 상태일 때 단일 실패 지점 문제를 피할 수 있도록 해보겠습니다.
