# **Arti: عميل Tor من الجيل التالي بلغة Rust**
![شعار Atri](https://gitlab.torproject.org/uploads/-/system/project/avatar/647/sliced-onions.png)

**Arti** هي مبادرة مشروع Tor لبناء عميل **Tor** من الجيل التالي باستخدام لغة البرمجة **Rust**. صُممت Arti لتكون معيارية، وقابلة للتضمين، وجاهزة للاستخدام الإنتاجي، بما يوفر تنفيذًا أكثر أمانًا وكفاءة لبروتوكولات إخفاء الهوية الخاصة بـ **Tor**. ومع **Arti الإصدار 1.4.0**، تم تقديم عدة تحديثات مهمة:

- **واجهة RPC جديدة** لتحسين التفاعل.
- أعمال تمهيدية لدعم **relay**.
- تحسينات في **مقاومة حجب الخدمة لخدمات onion من جهة الخدمة**.

يواصل هذا الإصدار جهود مشروع Tor لتقديم أمان وأداء ونمطية أفضل لمستخدمي Tor والمطورين.


---


## **تثبيت عميل Arti**

اتبع هذه الخطوات لتثبيت **Arti** وتشغيله كوكيل SOCKS على نظامك.

---

### **الخطوة 1: إعداد بيئة تطوير Rust**

قبل أن تتمكن من بناء Arti من الشيفرة المصدرية، يجب أن تكون لديك أحدث نسخة مستقرة من **Rust** مثبتة.

#### لتثبيت Rust:

1. قم بزيارة [موقع Rust الرسمي](https://www.rust-lang.org/).
2. اتبع تعليمات التثبيت الخاصة بنظام التشغيل لديك.
3. تحقق من التثبيت عبر تشغيل:
   
   ```sh
   rustc --version
   ```

سيؤكد ذلك أن أحدث نسخة مستقرة من Rust مثبتة على نظامك.

#### **ملاحظة لمستخدمي Windows**:
- يمكن تثبيت Rust على Windows عبر [**Rustup**](https://rustup.rs/)، وهي أداة لتثبيت toolchain. تأكد أيضًا من إعداد بيئة بناء متوافقة (قد تحتاج إلى **Visual Studio Build Tools** على Windows).
  
---

### **الخطوة 2: استنساخ مستودع Arti**

للحصول على أحدث إصدار من عميل Arti، ستحتاج إلى استنساخ المستودع من [**GitLab**](https://gitlab.torproject.org/tpo/core/arti).

#### الخطوات:
1. افتح الطرفية (Command Prompt أو PowerShell أو Git Bash على Windows).
2. شغّل الأمر التالي لاستنساخ المستودع:
   
   ```sh
   git clone https://gitlab.torproject.org/tpo/core/arti.git
   ```
4. انتقل إلى الدليل *arti* الذي تم إنشاؤه حديثًا:
   
   ```sh
   cd arti
   ```

سيؤدي ذلك إلى سحب الشيفرة المصدرية لـ Arti إلى جهازك المحلي.

---

### **الخطوة 3: بناء الملف التنفيذي لـ Arti**

بعد استنساخ المستودع، تحتاج إلى بناء Arti باستخدام **Cargo**، وهو مدير الحزم وأداة البناء الخاصة بـ Rust.

#### لبناء Arti:
1. في الطرفية، شغّل الأمر التالي:
   ```sh
   cargo build --release
   ```

يقوم هذا الأمر بترجمة شيفرة Arti وتحسينها للاستخدام الإنتاجي (العَلَم *--release*). سيتم إنشاء الملف التنفيذي في الدليل *target/release*.

#### موقع الملف التنفيذي المُترجَم:
- بعد البناء، سيكون الملف التنفيذي لـ Arti موجودًا في:  
  ```sh
  target/release/arti
  ```

يمكنك تشغيل هذا الملف التنفيذي مباشرة من الطرفية.

---

### **الخطوة 4: تشغيل وكيل Arti SOCKS**

لاستخدام Arti كوكيل SOCKS (والذي سيوجه حركة الإنترنت الخاصة بك عبر شبكة Tor)، تحتاج إلى بدء تشغيل الوكيل.

#### لبدء وكيل SOCKS:
1. شغّل الأمر التالي:
   ```sh
   ./target/release/arti proxy -p 9150
   ```

يؤدي هذا الأمر إلى تشغيل Arti كـ **وكيل SOCKS5** على **المنفذ 9150**، وهو المنفذ الافتراضي الذي يستخدمه Tor لحركة SOCKS.

---

### **الخطوة 5: تهيئة التطبيقات لاستخدام Arti**

بمجرد تشغيل Arti كوكيل SOCKS، تحتاج إلى تهيئة تطبيقاتك لاستخدامه في توجيه الحركة عبر شبكة Tor.

#### الخطوات:
1. في إعدادات تطبيقك (مثل متصفح الويب أو تطبيق الطرفية)، ابحث عن **إعدادات الوكيل**.
2. اضبط **وكيل SOCKS5** على *localhost:9150*.

سيؤدي ذلك إلى توجيه كل الحركة من تطبيقاتك عبر **شبكة Tor** باستخدام Arti كوسيط.

---

## **تكامل Arti مع شبكة Tor**

إليك مخططًا مبسطًا يوضح كيفية عمل Arti بالاقتران مع شبكة Tor:


```plaintext
[Application] --(SOCKS5)--> [Arti SOCKS Proxy] --(Tor Protocol)--> [Tor Network]
```

- يتصل **التطبيق** بـ **وكيل Arti SOCKS** باستخدام بروتوكول **SOCKS5**.
- بعد ذلك، يتواصل Arti مع **شبكة Tor**، مما يضمن إخفاء هوية حركتك أثناء مرورها عبر الشبكة.

## **مستودع GitLab والمساهمة**

إذا كنت مهتمًا بالمساهمة في تطوير **Arti**، يمكنك استكشاف الشيفرة والمساهمة عبر **GitLab**.

- **رابط المستودع**: [مستودع Arti على GitLab](https://gitlab.torproject.org/tpo/core/arti)
- **استنساخ المستودع**:
  ```sh
  git clone https://gitlab.torproject.org/tpo/core/arti.git
  ```

### **العمل عبر Fork والمساهمة**:
1. قم بإنشاء **Fork** للمستودع على GitLab (يتطلب ذلك حساب GitLab).
2. اربط المستودع المتفرع بإعدادك المحلي:
   ```sh
   git remote add _name_ git@gitlab.torproject.org:_name_/arti.git
   git fetch _name_
   ```
   استبدل *_name_* باسم المستخدم الخاص بك على GitLab.

3. **ادفع التغييرات** إلى الـ fork الخاص بك:
   ```sh
   git push _name_ main
   ```

4. **أنشئ Merge Request (MR)** على GitLab:
   انتقل إلى قسم Merge Request في fork الخاص بك على GitLab:
   ```plaintext
   https://gitlab.torproject.org/_name_/arti/-/merge_requests
   ```

### **إرشادات Merge Request**:
- **لا تقم بإعادة الأساس rebase ودمج الالتزامات squash أثناء المراجعة**.
- عند الحاجة، استخدم *fixup!* أو *squash!* لدمج الالتزامات تلقائيًا.
- احرص على **إضافة التزامات جديدة** بدلًا من دمجها أثناء دورة المراجعة.

---

### **ملاحظات إضافية**:

- **الملفات التنفيذية الجاهزة مسبقًا**: حتى الآن، لا توفر **Arti** ملفات تنفيذية جاهزة رسمية. يجب عليك بناء العميل من الشيفرة المصدرية كما هو موضح أعلاه.
- **المعرفة بـ Rust**: إذا كنت تساهم في Arti، فلاحظ أن قاعدة الشيفرة لا تزال تتطور، وقد تحدث تغييرات أو إعادة هيكلة مع إضافة ميزات جديدة.
