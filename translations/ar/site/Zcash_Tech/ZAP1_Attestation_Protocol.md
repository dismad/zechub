# بروتوكول المصادقة ZAP1

ZAP1 هو بروتوكول مصادقة مفتوح المصدر لـ Zcash. يكتب أحداث دورة حياة منظَّمة إلى شجرة Merkle باستخدام BLAKE2b ويُثبّت جذر الشجرة على السلسلة عبر مذكرات Orchard المحمية. يمكن التحقق من الإثباتات علنًا. وتبقى بيانات الأحداث خاصة.

## كيف يعمل

يسجّل المشغّلون أنواع الأحداث (عمليات النشر، والمدفوعات، والتحويلات، وما إلى ذلك) ويرسلونها إلى مثيل ZAP1. ينتج عن كل حدث تجزئة ورقة باستخدام BLAKE2b-256 مع فصل المجال. وتتراكم الأوراق في شجرة Merkle. وعند بلوغ حدّ معيّن، يُرمَّز جذر الشجرة كمذكرة ZAP1:09 ويُثبَّت على Zcash في معاملة محمية.

يمكن لأي شخص يمتلك تجزئة ورقة التحقق من المسار الكامل من الورقة إلى الجذر ثم إلى التثبيت على السلسلة، من دون الحاجة إلى الثقة بالمشغّل.

## الخصائص الرئيسية

- **غير مرتبط بتطبيق معيّن**: يمكن لأي مشغّل Zcash تعريف أنواع أحداثه الخاصة وسلاسل التخصيص الخاصة به
- **يحافظ على الخصوصية**: تُجزَّأ حمولات الأحداث قبل تثبيتها. ولا يُكتب على السلسلة سوى التجزئات.
- **قابل للتحقق بشكل مستقل**: لا يتطلب التحقق سوى حزمة الإثبات وإمكانية الوصول إلى السلسلة. ولا حاجة إلى الثقة بالمشغّل.
- **متوافق مع ZIP 302**: يتقارب ZAP1 نحو partType ضمن ZIP 302 لحمولة المصادقة

## ما هو موجود حاليًا

- تنفيذ مرجعي (Rust، بترخيص MIT)
- مجموعة تطوير تحقق على crates.io ‏(Rust + 83KB WASM)
- مجموعة تطوير JavaScript على npm
- مفكك ترميز مذكرات شامل (يحدّد ZAP1، وZIP 302 TVLV، والنص، والثنائي، والمذكرات الفارغة)
- مجموعة مطابقة تتضمن 29 فحصًا لواجهة API و14 فحصًا للبروتوكول
- تصميم توقيع عتبي FROST ‏2 من 3 لبثّ نقاط التثبيت متعددة الأطراف
- مسودة ZIP في طلب السحب PR #1243 قيد المراجعة
- 4 نقاط تثبيت على الشبكة الرئيسية مع 14 ورقة حتى مارس 2026

## البنية المعمارية

```
Your app  -->  ZAP1 API  -->  Merkle tree  -->  Zcash anchor
                  |                                    |
             event types                         shielded memo
          (DEPLOYMENT, etc)                    (ZAP1:09:{root})
```

يشغّل كل مشغّل مثيل ZAP1 خاصًا به مع مفاتيحه الخاصة وشجرة Merkle ونقاط التثبيت الخاصة به. ولا توجد حالة مشتركة بين المشغّلين.
