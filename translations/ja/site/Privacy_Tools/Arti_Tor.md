# **Arti: Rustで構築された次世代のTorクライアント**
![Atri ロゴ](https://gitlab.torproject.org/uploads/-/system/project/avatar/647/sliced-onions.png)

**Arti**は、**Torプロジェクト**が**Rust**プログラミング言語を使用して構築する次世代の**Tor**クライアントを開発するためのイニシアチブです。**Arti**はモジュール式で埋め込み可能であり、本番環境での使用に適しており、**Tor**匿名プロトコルをより安全かつ効率的に実装しています。**Artiバージョン1.4.0**では、以下の重要なアップデートが導入されています：

- **新しいRPCインターフェース**による相互作用の向上。
- **リレーサポート**用の準備作業。
- **サービス側オンイオンサービスDoS耐性**の改善。

このリリースは、Torユーザーおよび開発者にとってセキュリティ、パフォーマンス、モジュール性を向上させるTorプロジェクトの取り組みの一環です。


---


## **Artiクライアントのインストール**

以下の手順に従って、システム上で**Arti**をSOCKSプロキシとしてインストールして実行してください。

---

### **ステップ1: Rust開発環境のセットアップ**

ソースからArtiをビルドするには、最新の安定版の**Rust**がインストールされている必要があります。

#### Rustのインストール方法:

1. 公式[**Rustウェブサイト**](https://www.rust-lang.org/)にアクセスしてください。
2. ご使用のオペレーティングシステム用のインストール手順に従ってください。
3. インストールを確認するために以下のコマンドを実行してください：

   ```sh
   rustc --version
   ```

これにより、システム上に最新の安定版Rustがインストールされていることが確認できます。

#### **Windowsユーザー向けの注意事項**:
- Windowsでは[**Rustup**](https://rustup.rs/)を使用してRustをインストールできます。これはツールチェーンインストーラーです。また、互換性のあるビルド環境（Windowsでは**Visual Studio Build Tools**が必要）も設定してください。

---

### **ステップ2: Artiリポジトリのクローン**

最新版のArtiクライアントを取得するには、[**GitLab**](https://gitlab.torproject.org/tpo/core/arti)からリポジトリをクローンする必要があります。

#### 手順:
1. テーミナル（Windowsではコマンドプロンプト、PowerShell、またはGit Bash）を開きます。
2. 以下のコマンドを実行してリポジトリをクローンします：

   ```sh
   git clone https://gitlab.torproject.org/tpo/core/arti.git
   ```
4. 新しく作成された*arti*ディレクトリに移動します：

   ```sh
   cd arti
   ```

これにより、Artiのソースコードがローカルマシンに引き込まれます。

---

### **ステップ3: Artiバイナリのビルド**

リポジトリをクローンした後は、**Cargo**を使用してArtiをビルドする必要があります。CargoはRustのパッケージ管理およびビルドツールです。

#### Artiのビルド方法:
1. テーミナルで以下のコマンドを実行します：
   ```sh
   cargo build --release
   ```

このコマンドにより、Artiコードがコンパイルされ、本番環境用に最適化されます（*--release*フラグ）。バイナリは*target/release*ディレクトリ内に作成されます。

#### ビルドされたバイナリの場所:
- ビルド後、Artiバイナリは以下の場所に配置されます：
  ```sh
  target/release/arti
  ```

このバイナリを直接テキストエディタから実行できます。

---

### **ステップ4: Arti SOCKSプロキシの起動**

ArtiをSOCKSプロキシとして使用するには（インターネットトラフィックをTorネットワーク経由でルーティングするために）、プロキシを起動する必要があります。

#### SOCKSプロキシの起動方法:
1. 以下のコマンドを実行します：
   ```sh
   ./target/release/arti proxy -p 9150
   ```

このコマンドにより、Artiが**ポート9150**で**SOCKS5プロキシ**として起動し、Torのデフォルトポートとして使用されます。

---

### **ステップ5: アプリケーションをArtiに設定**

ArtiがSOCKSプロキシとして動作している後は、アプリケーションを設定して、Torネットワーク経由でトラフィックをルーティングするようにします。

#### 手順:
1. アプリケーションの設定（例: ブラウザやターミナルアプリ）を開き、**プロキシ設定**を探してください。
2. **SOCKS5プロキシ**を*localhost:9150*に設定してください。

これにより、すべてのアプリケーションからのトラフィックがArtiを通じて**Torネットワーク**経由でルーティングされます。

---

## **ArtiとTorネットワークとの統合**

以下は、ArtiがTorネットワークとどのように連携するかを示した簡略図です：

```plaintext
[アプリケーション] --(SOCKS5)--> [Arti SOCKSプロキシ] --(Torプロトコル)--> [Torネットワーク]
```

- **アプリケーション**は、**SOCKS5プロトコル**を使用して**Arti SOCKSプロキシ**に接続します。
- Artiはその後、**Torネットワーク**と通信し、トラフィックがネットワークを通過する際に匿名化されるようにします。

## **GitLabリポジトリと貢献**

**Arti**の開発に貢献したい場合は、コードを確認し、**GitLab**を通じて貢献してください。

- **リポジトリリンク**: [Arti GitLabリポジトリ](https://gitlab.torproject.org/tpo/core/arti)
- **リポジトリのクローン**:
  ```sh
  git clone https://gitlab.torproject.org/tpo/core/arti.git
  ```

### **フォークと貢献**:
1. GitLabでリポジトリをフォークしてください（GitLabアカウントが必要です）。
2. フォークしたリポジトリをローカル環境にリンクします：
   ```sh
   git remote add _name_ git@gitlab.torproject.org:_name_/arti.git
   git fetch _name_
   ```
   *name*はGitLabのユーザー名に置き換えてください。

3. **変更をプッシュ**してフォークに送信します：
   ```sh
   git push _name_ main
   ```

4. GitLabで**マージリクエスト（MR）**を作成してください：
   フォークしたGitLabリポジトリのMerge Requestセクションへアクセスしてください：
   ```plaintext
   https://gitlab.torproject.org/_name_/arti/-/merge_requests
   ```

### **マージリクエストガイドライン**:
- **レビュー中にコミットを再ベースまたはスカッシュしないでください**。
- 必要に応じて、*fixup!*や*squash!*を使用して自動スカッシュを行ってください。
- レビュー期間中は、**新しいコミットの追加**を目指してください。
