- 全ての取引でシールドされたアドレスを使用する
- 財布のアイデンティティを公開してはならない
- 準備可能な支払いを避けて情報源を保護する

ジャーナリストは以下に直面します：
- 監視
- 情報源の暴露リスク
- 敵対者による財務追跡

透明な暗号通貨を使用すると以下の情報が明らかになります：
- 誰があなたに支払ったか
- あなたが誰に支払ったか
- いつ、どのくらい支払ったか

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-lock.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="ツールボックスicon"/> なぜZcashなのか？

Zcashは以下を提供します：
- 強力な財務プライバシー
- 敏感な取引の保護
- ジャーナリストと情報源の関連性リスクの低減

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-toolbox.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="ツールボックスicon"/> 必要なもの

- シールド機能をサポートするZcash財布
- ブロックチェーン外でのセキュアな通信チャネル
- 操作の自律性

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="ステップicon"/> ステップバイステップガイド

### ステップ1: 専用の財布を作成する

- 敏感な作業専用の財布を使用する
- 個人的な取引と混ぜない

### ステップ2: シールドされたアドレスのみを使用する

- いつもzアドレスを使って取引を行う
- 透明な取引を完全に避ける

### ステップ3: 情報源への支払いを保護する

情報源に支払う場合:
- シールドされた取引（z → z）を使用する
- 予測可能なタイミングパターンを避ける
- アドレスの再利用を行わない

### ステップ4: メモの使用には注意する

- メモに機密情報を含めない
- 必要であれば中立的または暗号化された参照を使用する

### ステップ5: 操作セキュリティを維持する

- 財布を自分のアイデンティティと関連付けない
- 取引の確認を公開しない
- 通信には別のチャネルを使用する
