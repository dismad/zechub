# Zcash バリューポール

## TL;DR

- 現在のZcashには **4つのバリューポール** があります: Sprout（レガシ）、Sapling、Orchard、およびTransparent。
- **Orchard** は現在、統合アドレス（u1...）で使用されている主なシャドウドポールです。
- **Sapling**（`zs`で始まるzアドレス）は広くサポートされており、多くのシャドウドZECを保護しています。
- **Transparent** アドレス（t...）は取引のプライバシーを提供せず、ビットコインと同様に動作します。
- **Sprout** はレガシーシャドウドポールであり、現在では活用されていません。
- 将来のシャドウドポールとして、**Ironwood** が提案されており、シャドウドZECの供給の整合性に対する信頼を強化しつつプライバシーを維持することを目指しています。
- 最も強いプライバシー保証を得るためには、可能な限り **shielded-to-shielded（z → z）** 取引を使用することが推奨されます。


<br/>

## Zcash バリューポールの理解

Zcashは、資金をそれぞれ独自の会計システムとして分離した「バリューポール」として管理します。各ポールには独自の暗号技術ルールとプライバシー特性があり、プロトコルはそれらの間で移動する総価値を追跡しています。

現在、ネットワークには4つの主要なバリューポールがあります：

- Transparent — 公開され、チェーン上では完全に見える。
- Sapling — 最初に広く採用された現代的なシャドウドポール。
- Orchard — 統合アドレスとともに導入された現在の主なシャドウドポール。
- Sprout — 2016年にZcashがローンチされた際のオリジナルのシャドウドポール。


Zcashが進化するにつれて、セキュリティ、プライバシー、使いやすさ、監査可能性を向上させながら既存の資金と互換性を保つ新しいシャドウドポールが導入される可能性があります。

## シャドウドポール


1. <h3 id="orchard" class="text-3xl font-bold my-4">Orchard ポール</h3>

![img2](https://github.com/user-attachments/assets/a672e001-6dbc-4e76-ab31-0ed7d7d2ff72)
図2: 2025年10月時点でのOrchardポールを示したチャート

<br/>

Orchard シャドウドポールは、2022年5月31日にNU5ネットワークアップグレードの一環としてアクティブ化されました。Orchardは信頼設定の必要性を排除する新しいシャドウドプロトコルを導入し、統合アドレス（UA）で使用される主なシャドウドポールとなりました。

Orchardは、取引メタデータの漏洩を減らし、従来のシャドウドインプットとアウトプットではなくアクションに基づくより柔軟な取引モデルを導入することで、使いやすさ、効率性、プライバシーを大幅に改善しました。

現在、OrchardはZcashの主なシャドウドポールです。しかし、コミュニティは将来的な移行先として、シャドウドZEC供給の整合性に関する追加の保証を提供しつつZcashのプライバシー保証を維持する新しいシャドウドポールであるIronwoodへの移行を検討しています。

[Zcash シャドウドウォレット](/site/Using_Zcash/Wallets)は現在、Orchardをサポートしています。 

____

2. <h3 id="sapling" class="text-3xl font-bold my-4">Sapling ポール</h3>

![img3](https://github.com/user-attachments/assets/b1c6bb71-9356-45eb-8e4a-19d7cf1790ae)
図3: 2025年10月時点でのSaplingポールを示したチャート

<br/>

[Zcash Sapling](https://z.cash/upgrade/sapling)は、2018年10月28日に導入されたZcashプロトコルのアップグレードです。これは、プライバシー、効率性、使いやすさにおいていくつかの制限があった以前のバージョン（Sprout）よりも大幅に改善されています。

一部のアップグレードには、シャドウドアドレスのパフォーマンス向上、ユーザーが送金および受信取引を確認できるようになるための改良されたビューイングキー、ハードウェアウォレットでのトランザクション署名中に独立したゼロ知識キーやなどがあります。

Zcash Saplingは、Sproutシリーズに比べてわずか数秒でプライベートな取引が可能になります。トランザクションのシャドウド化により、第三者が取引をリンクしたり、転送されるZECの金額を特定したりすることが不可能になります。Saplingはまた、ユーザーにとってプライベートなトランザクション生成に必要な計算リソースを削減し、より使いやすくなりました。

Saplingウォレットアドレスは「zs」で始まります。これはすべてのサポートされているZcashシャドウドウォレット（YWallet、Zingo Wallet Nighthawkなど）で内蔵されたSaplingアドレスとして観測できます。Zcash Saplingはプライバシーと効率性において技術的な大きな進展をもたらし、プライバシーとセキュリティを重視するユーザーにとって実用的かつ効果的な暗号通貨となっています。

____

3. <h3 id="sprout" class="text-3xl font-bold my-4">Sprout ポール</h3>

![img4](https://github.com/user-attachments/assets/956eceed-f4d6-4087-99d0-32a770449dda)
図4: 2025年10月時点でのSproutポールを示したチャート

Sproutは、最初のオープンで権限なきゼロ知識プライバシープロトコルであり、2016年10月28日にローンチされました。

Sproutアドレスは常に「zc」で始まります。これはソフトウェアが若く、芽生えているブロックチェーンであることを強調するために「Sprout」と名付けられました。

Sproutは[Zcash slow start Mining](https://electriccoin.co/blog/slow-start-and-mining-ecosystem/)の初期ツールとして使用され、ZECとマイナーへのブロック報酬の配布をもたらしました。

Zcashエコシステムが拡大し、シャドウドトランザクションの数が増えるにつれて、Sproutシリーズはユーザーのプライバシー、トランザクションのスケーラビリティおよび処理において限界があり、効率性が低くなることが観測されました。これによりネットワークの変更とSaplingアップグレードが行われました。

---
4. <h3 id="transparent" class="text-3xl font-bold my-4">Transparent ポール</h3>
<br/>

![img5](https://github.com/user-attachments/assets/01de2907-b62d-4421-83d7-ea4908faa828)
図5: 2025年10月時点でのTransparentポールを示したチャート

<br/>

Zcash Transparent ポールは非シャドウドで非プライベートです。ZcashのTransparentウォレットアドレスは「t」で始まり、このタイプのアドレスを使用してトランザクションを行うとプライバシーが非常に低くなります。

ZcashのTransparentトランザクションはビットコインのトランザクションに類似しており、マルチ署名トランザクションをサポートし、標準的な公開アドレスを使用します。

Zcash Transparentは主に中央集権型取引所によって使用され、ユーザー間でZECを送金および受信する際の高い透明性とネットワーク確認を保証するために用いられます。

また、Zcash Shieldedアドレスがトランザクション中に高いプライバシーを提供することに注意が必要ですが、トランザクション処理にはより多くの計算リソースが必要です。そのため、プライバシーが不要なトランザクションでは一部のユーザーがTransparentアドレスを使用することがあります。

<br/>

## ポール転送の推奨実践

Zcashネットワーク上で取引中に高いレベルのプライバシーを確保するためには、以下の実践を推奨します；

Zcashブロックチェーン上の「z to z」ウォレット間で発生するトランザクションは主にシャドウドであり、高レベルのプライバシーが生成されるため、プライベートトランザクションと呼ばれます。これはプライバシーが必要な場合に送金および受信を行う際の最良かつ最も推奨される方法です。

---

「Z-address」から「T-address」へZECを送る行為は、デシャドウドトランザクションと呼ばれています。このタイプのトランザクションではプライバシーが常に高くなるわけではなく、透明アドレスへのZEC送信によりブロックチェーン上に一部の情報が見えるようになるためです。高いプライバシーが必要な場合、デシャドウドトランザクションは常に推奨されません。

---

Transparentアドレス（T-address）からZ-addressへZECを転送する行為はShieldingと呼ばれます。このタイプのトランザクションではz-zトランザクションに比べてプライバシーが常に高くなるわけではありませんが、プライバシーが必要な場合にも推奨されます。

---

Zcashネットワーク上のTransparentアドレス（T-address）から別のTransparentアドレス（T-address）へZECを送る行為（T-Tトランザクション）はビットコインのトランザクションと非常に類似しており、このためZcash上でのT-Tトランザクションは常に「パブリックトランザクション」と呼ばれます。これは、送信者および受信者のトランザクション詳細が公開されるためプライバシーのレベルが非常に低くなります。

ほとんどの暗号通貨中央集権型取引所はZcashブロックチェーン上での取引においてTransparentアドレス（T-address）を使用しますが、このタイプのトランザクション（T-T）にはプライバシー性はありません。


<br/>

## 未来：Ironwood ポール

Zcashコミュニティは現在、提案されているシャドウドポールであるIronwoodを評価しています。

IronwoodはOrchardの証明システムに最近発見され、修正された脆弱性に対処するために設計されています。この脆弱性が実際に悪用されたという証拠はありませんが、IronwoodはOrchardから新たに作成されたシャドウドポールへの制御された移行を可能にし、追加の保証層を提供します。

目的はZcashのプライバシーを置き換えることではなく、シャドウドZEC供給の整合性に対する信頼を強化することです。

## 提案内容：

1. 新しいシャドウド活動は段階的にIronwoodに移行されます。
2. 既存のOrchard資金はプライバシーを保ちながら移動できます。
3. 公開されるターンスタイル会計により、すべてのシャドウド資金が完全に裏付けられていることをより強固な証拠として提供します。
4. ユーザーはZcashから期待されている同じプライバシー保護を維持します。

<br/>
将来的なネットワークアップグレードを通じてアクティブ化された場合、Ironwoodは既存のシャドウド資金と互換性を保ちながら、Zcashのシャドウドエコシステムの次の世代となるでしょう。


<br/>

## 避けたい一般的なミス

- **t-addressからt-addressへの送金** — 完全に公開され、プライバシーなし。常に資金をまずシャドウド化してください。
- **SaplingとOrchardアドレスの混同** — Saplingアドレスは`zs`で始まり、Orchard/統合アドレスは`u1`で始まる
- **Sproutポールに資金を残す** — Sproutは非推奨です。資金をOrchardへ移行してください。
- **t → z（シャドウド化）が完全にプライベートであると仮定する** — シャドウド化そのものはチェーン上に見えるが、内容自体は見えない
