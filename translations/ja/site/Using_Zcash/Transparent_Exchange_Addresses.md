# ZcashのTEXアドレスとは？

ZcashのTEXアドレスは、受信用のユニークなタイプのアドレスを表します。"Transparent Exchange"（透明取引所）を意味する略語で、単一のp2pkh透明アドレスの**ユニーク**かつ統合型（bech32m）エンコードです。

その唯一の目的は、互換性のあるウォレットに、**透明専用（T → T）取引**を実行するよう通知することです。

論理的な流れは以下の通りです：TEXアドレスが検出されると、互換性のあるウォレットはそれをデコードして、中に含まれる透明受信者情報を取得します。ウォレットはその後、txに必要な資金をシールドプールからユーザー自身が制御する一時的な透明アドレス（Z → T）へ送金し、その資金をTEXアドレスのデコードされた透明受信者（T → T）へ送ります。

TEXアドレスに関する技術提案は、Zcashの[ZIP 320](https://zips.z.cash/zip-0320)に記載されており、透明アドレスから資金を受け取るための専用のアドレスタイプを定義しています。

![TEX](https://i.ibb.co/8m7HPqV/ZashiTex.png)

TEXアドレスは広く採用されていませんが、Zcashユーザーは最終的にそれらを使用する必要があるかもしれません。

## TEXアドレスが必要になるのはいつですか？

### 透明アドレスに資金を送る際、直接送金をサポートしていないウォレットを使用している場合、**TEXアドレスが必要です**。
一部のウォレットでは、透明アドレスへの直接送金が許可されていないため、**受信者がTEX相当のアドレスを提供しない可能性があります**。このため、**透明アドレスからTEXアドレスへの変換**が必要になることがあります。これはzip-320に記載されたリファレンス実装を手動で実行することで達成できます。**透明→TEX変換器**のホストインスタンスは[こちら](https://690e9524c66a3ecac5d54eff--jade-brioche-873777.netlify.app/)で確認できます。

### 中央集権的な取引所に資金を送る際、**その資金が透明なソースから来ている必要がある場合**、TEXアドレスが必要です。
現在では、[Binance](https://www.binance.com/)のみがTEXアドレスを使用している中央集権的な取引所であり（これがTEXの作成の主な理由でもあります）、TEXアドレスは互換性のあるウォレットに、そのアドレスへ送金されるすべての資金が透明であることを通知し、シールドされた値をそのアドレスへ送金することを排除します。
もしBinanceのような取引所が送金された値を拒否した場合、それを元のアドレスに戻す手段を持っています。また、Binanceのような企業にとっても、政府や他の権限から課せられた法律や規制に従うことを助けます。
