# IPFSとBraveブラウザを使ったP2Pファイル共有

IPFSを使用して、私たちの間で大容量メディアファイルをPeer to Peer方式で共有できます。

では問題点は何か？：IPFSゲートウェイは大容量ファイルには対応しておらず、タイムアウトします。

**要件**: [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) + [Brave Browser](https://brave.com/)

tldr, ビデオは最後にあります。

ZF AV Clubの間で大容量メディアファイルを共有し、それを皆様のために記録することができるのは、とても楽しくてすごくクールです！！

*PLUS*, 今回はProtocol LabsのIPFSチームと直接協力してこのユースケースを開発します。
これはdWeb Campでのつながりから生まれたもう一つの利点です！

受信側のマシンにIPFSがインストールされていない、またはBrave以外のブラウザを使用している場合：ファイルをロードするまでゲートウェイを待つ必要があります。*大容量ファイルでは504エラーが返されます。*

次は、他の人のファイルを自分のマシンでピン留めして、オリジナルノードがオフラインになったときに単一点の障害問題を回避する方法です。
