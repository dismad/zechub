# Zcashウォレットの同期

## 要点

* シールドされたZcashトランザクションはその詳細を隠すため、サーバーはBitcoinやEthereumのような透明なコインと同じ方法でウォレット残高を単純に照会できません。
* ライトウォレットは、専用サーバー（lightwalletd）から小さな「コンパクトブロック」をダウンロードし、自分の秘密鍵を使って関連データを自ら復号します。
* それらのブロックを復号して処理するには時間がかかるため、ウォレットはより高速な同期方法を使って、資金をより早く利用できるようにしています。
* 主なアプローチ: Warp Sync（YWallet）、Spend-before-sync（Zcash Mobile Wallet SDK V2）、Blaze Sync（Zecwallet）、そして提案中のDAGSync。
* これらの方法は一般に、同期を高速化する代わりに、より多くのメモリや処理能力を必要とします。

## 基本的な説明

### Zcashの同期の仕組み

Zcashは、認可されていない第三者からトランザクションの詳細を隠すためにゼロ知識証明を使用しています。このプライバシーにより、ライトウォレットにとって同期はより難しくなります。なぜなら、ライトウォレットは完全なブロックチェーンをローカルに保存せず、代わりに必要な情報をサーバーに頼るからです。BitcoinやEthereumでは、サーバーがブロックチェーンにインデックスを作成し、アカウントデータを素早く返せます。しかしZcashでは、サーバーはトランザクションの詳細を見ることができません。では、ライトウォレットはブロックチェーン全体を自分でダウンロードして復号しなくても、どうやって残高と履歴を同期できるのでしょうか？

Zcashはこの問題を、複数のアプローチを組み合わせることで解決しています。Zcashには、フルノードからデータを絞り込み、トランザクション識別に必要なものだけを保持する専用サーバー lightwalletd があります。このデータはコンパクトブロックと呼ばれ、元のブロックよりもはるかに小さいものです。ライトウォレットはまず、このコンパクトブロックをlightwalletdサーバーからダウンロードし、その後で自分の秘密鍵を使って復号します。

こうしたコンパクトブロックであっても、復号と処理にはかなりの時間がかかることがあります。特に、1ブロックあたりのトランザクション数が多い場合はなおさらです。そのためウォレットは、同期を高速化し、できるだけ早く資金を使えるようにするために、さまざまな方法を採用しています。

## 視覚的なイメージ / たとえ話

ブロックチェーンを、鍵のかかった箱でいっぱいの巨大な郵便室だと考えてみてください。透明なコインなら、郵便室の係員がラベルを読んで、どの箱があなたのものかを即座に教えてくれます。Zcashではラベルが隠されているため、あなたのウォレットは自分の鍵を持って静かに箱を確かめ、自分で開けられるものを探さなければなりません。以下の同期方法は、それらの箱をより速く確認するための異なる戦略です。

## 詳細解説

### Warp Sync

Warp syncはYWalletの機能で、各コンパクトブロックを復号して処理する中間ステップを飛ばし、最終結果に直接たどり着きます。

そのために、各ステップを順に実行するのではなく、数学と暗号技術を使って最終結果を計算します。

Warp syncは1秒あたり数千ブロックを処理でき、通常の同期方法よりはるかに高速です。つまりYWalletのユーザーは、アカウントに数十万件のトランザクションや受信ノートがあっても、高速でスムーズな動作を利用できます。

このステップ省略技術に加えて、YWalletは複数のブロックを同時に処理でき、利用可能なハードウェア全体に負荷を分散することで、処理をさらに高速化します。

詳しくは[Warp Sync](https://ywallet.app/warp/)をご覧ください

### Spend-before-sync

Spend-before-syncは、Zcash Mobile Wallet SDK V2の新機能で、ユーザーがウォレットを開いたら完全な同期を待たずに即座に資金を使えるようにするものです。この機能により、ウォレットの利用可能残高の発見が高速化され、ユーザー体験が向上します。

Spend-before-syncは、lightwalletdサーバーからのブロックを非線形な順序で処理するコンパクトブロック同期アルゴリズムを使って動作します。つまり、1つのブロックを完全に処理してから次へ進むのを待つ代わりに、ウォレットは少し多くのメモリと処理能力を使って、ブロックチェーンの異なる部分をスキャンできます。通常は異なる範囲をスキャンし、古いブロックがダウンロード・処理されている間に、新しいトランザクションを探します。最近の未使用ノートが見つかれば、それはすぐに利用可能になります。

### Blaze Sync

Zecwalletチームによって開発されたBlaze syncは、最新かつ最も高いブロックから始めて後方へ向かってスキャンする、ライトウォレット向けの同期アルゴリズムです。

これにより、ウォレットは受信ノートより先に使用済みノートを見つけられる一方で、以前は未使用だったノートを、完全な同期プロセスの終了を待たずに利用可能にできます。

さらに、同期の各コンポーネント同士を切り離し、ブロックのダウンロード、試行復号、witnessの更新を並列処理することで、Out-of-Order Syncを利用します。これにはより多くのメモリとCPUリソースが必要ですが、同期速度を5倍に向上させます。

### DAGSync

DAGSyncは、同期を高速化することでZcashシールドウォレットのユーザー体験を改善することを目指した、提案中の同期アルゴリズムです。

これは、[有向非巡回グラフ（DAG）](https://words.str4d.xyz/dagsync-graph-aware-zcash-wallets/)を使って、Zcashウォレットにおけるノート、witness、nullifier間の依存関係を表現します。

DAGはノードと辺から成るデータ構造で、各辺には2つのノード間の関係を示す方向があります。DAGにはサイクルがなく、つまり、あるノードから始めて辺をたどり、同じノードに戻ることはできません。

## 実際の意味合い

興味深いことに、これらの仕組みはすべて、Zcash Securityが[Scalable Private Messaging](https://zecsec.com/posts/scalable-private-money-needs-scalable-private-messaging/)に関する投稿と、プライベートな決済システムとの関係の中で提起した問いに対処することを目指しています。中には、アドレス固有のデータを除き、すべてのメモデータをサーバーからダウンロードするという追加の一歩を踏み出すものもあり、わずかな追加リソースを代償にプライバシーを高めています。

また、Zcash Foundationも、ライトウォレットの性能を改善するための他の選択肢を検討してきました。その一例が[Oblivious Message Retrieval (OMR)](https://zfnd.org/oblivious-message-retrieval/)であり、これは財団が「最近Zcashウォレットのユーザーに影響を与えている性能上の問題に対して、潜在的な解決策を提供できるかどうかを判断するため」に研究している構成です。

## よくある誤解

**lightwalletdサーバーがあなたの残高を知っていると思い込むこと。** サーバーが配信するのはコンパクトブロックだけであり、ウォレットはそれを自分自身の鍵でローカルに復号し解釈します。

**同期を早く止めすぎること。** 一部の方法では、完全な同期が終わる前に最近の利用可能な資金を使えるようになりますが、古い履歴やノートの処理はまだ進行中である可能性があります。

**Zcashの同期を透明チェーンの同期と直接比較すること。** 遅い経路は欠陥ではなく、プライバシーを守るためのコストかもしれません。ウォレットは、本来なら公開コインのサーバーがあなたのアカウントを公開状態で読み取って行う作業を、自分で処理しているのです。
