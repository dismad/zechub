# Zcashウォレットの同期

## 要点

* シールドされた Zcash トランザクションは詳細を隠すため、Bitcoin や Ethereum のような透明なコインと同じように、サーバーがウォレットの残高を単純に照会することはできません。
* ライトウォレットは、専用サーバー（lightwalletd）から小さな「コンパクトブロック」をダウンロードし、自分の秘密鍵を使って関連データを自ら復号します。
* それらのブロックの復号と処理には時間がかかるため、ウォレットはより高速な同期方法を用いて、資金をより早く使えるようにしています。
* 主なアプローチ: Warp Sync（YWallet）、Spend-before-sync（Zcash Mobile Wallet SDK V2）、Blaze Sync（Zecwallet）、そして提案中の DAGSync。
* これらの方法は一般に、より高速な同期のために追加のメモリや処理能力を引き換えにします。

## 基本的な説明

### Zcash の同期の仕組み

Zcash は、ゼロ知識証明を使って、許可されていない第三者からトランザクションの詳細を保護します。このプライバシーにより、ライトウォレットの同期は難しくなります。ライトウォレットはブロックチェーン全体をローカルに保存せず、代わりに必要な情報をサーバーに依存しているからです。Bitcoin や Ethereum では、サーバーがブロックチェーンをインデックス化し、アカウントデータを素早く返せます。しかし Zcash では、サーバーはトランザクションの詳細を見ることができません。では、ライトウォレットはブロックチェーン全体を自分でダウンロードして復号しなくても、どうやって残高と履歴を同期できるのでしょうか？

Zcash は、この問題を複数のアプローチを組み合わせることで解決しています。lightwalletd という専用サーバーがあり、フルノードからデータをフィルタリングして、トランザクション識別に必要なものだけを保持します。このデータはコンパクトブロックと呼ばれ、元のブロックよりもはるかに小さいものです。ライトウォレットは、まずこれらのコンパクトブロックを lightwalletd サーバーからダウンロードし、その後で秘密鍵を使って復号します。

これらのコンパクトブロックであっても、復号と処理にはかなりの時間がかかることがあります。特に、1ブロックあたりのトランザクション数が多い場合はなおさらです。そのため、ウォレットはさまざまな方法を使って同期を高速化し、できるだけ早く資金を使えるようにしています。

## 視覚的な説明 / たとえ

ブロックチェーンを、鍵のかかった箱でいっぱいの巨大な郵便室だと考えてみてください。透明なコインなら、郵便室の係員はラベルを読んで、どの箱があなたのものかをすぐに教えてくれます。Zcash ではラベルが隠されているため、あなたのウォレットは自分の鍵を使って静かに箱を確かめ、開けられるものを見つけなければなりません。以下の同期方法は、その箱をより速く確認するための異なる戦略です。

## 詳しく見る

### Warp Sync

Warp sync は YWallet の機能で、各コンパクトブロックの復号と処理という中間ステップを飛ばし、最終結果に直接たどり着きます。

そのために、数学と暗号技術を使って、各ステップを順番にたどることなく最終結果を計算します。

Warp sync は 1 秒あたり数千ブロックを処理でき、通常の同期方法よりもはるかに高速です。つまり、YWallet ユーザーは、自分のアカウントに数十万件のトランザクションや受信ノートがあっても、高速でスムーズな動作を享受できます。

このステップ省略の技術に加えて、YWallet は複数のブロックを同時に処理できるため、利用可能なハードウェア全体に負荷を分散し、処理をさらに高速化します。

詳しくは [Warp Sync](https://ywallet.app/warp/) をご覧ください

### Spend-before-sync

Spend-before-sync は Zcash Mobile Wallet SDK V2 の新機能で、ユーザーがウォレットを開いた瞬間に、ウォレット全体の同期が完了するのを待たずに資金をすぐ使えるようにします。この機能は、ウォレットの使用可能残高の発見を高速化し、ユーザー体験を向上させます。

Spend-before-sync は、lightwalletd サーバーからのブロックを非線形の順序で処理する compact-blocks 同期アルゴリズムを使って動作します。つまり、1つのブロックを完全に処理してから次に進むのを待つ代わりに、ウォレットは少し多めのメモリと処理能力を使って、ブロックチェーンの異なるセクションをスキャンできます。通常は異なる範囲をスキャンし、古いブロックをダウンロードして処理している間に、より新しいトランザクションを探します。最近の未使用ノートが見つかれば、それは直ちに利用可能になります。

### Blaze Sync

Zecwallet チームによって開発された Blaze sync は、最新かつ最も高いブロックから始めて、そこから後方へ向かってブロックチェーンをスキャンする、ライトウォレット向けの同期アルゴリズムです。

これにより、ウォレットは受信ノートより前に使用済みノートを見つけられる一方で、以前は未使用だったノートを、完全な同期プロセスの終了を待たずに利用可能にできます。

そのうえ、同期の各構成要素同士を分離し、ブロックのダウンロード、試行復号、witness の更新を並列に処理することで、Out-of-Order Sync も利用します。これはより多くのメモリと CPU リソースを必要としますが、同期速度を 5 倍向上させます。

### DAGSync

DAGSync は、同期を高速化することで、Zcash のシールドウォレットのユーザー体験向上を目指す提案中の同期アルゴリズムです。

これは、Zcash ウォレット内のノート、witness、nullifier の依存関係を表現するために [Directed Acyclic Graph (DAG)](https://words.str4d.xyz/dagsync-graph-aware-zcash-wallets/) を使用します。

DAG はノードとエッジから成るデータ構造で、各エッジには2つのノード間の関係を示す方向があります。DAG にはサイクルがなく、つまり、あるノードから始めてエッジをたどり、同じノードに戻ってくることはできません。

## 実際の意味合い

興味深いことに、これらすべての仕組みは、Zcash Security が [Scalable Private Messaging](https://zecsec.com/posts/scalable-private-money-needs-scalable-private-messaging/) に関する投稿で提起した問題と、プライベート決済システムとの関係に対処することを目指しています。中には、アドレスに固有のデータを除き、すべてのメモデータをサーバーからダウンロードするところまで進めるものもあり、少し余分なリソースを代償にプライバシーを高めています。

また、Zcash Foundation は、ライトウォレットの性能を改善するための別の代替案も検討してきました。その一例が [Oblivious Message Retrieval (OMR)](https://zfnd.org/oblivious-message-retrieval/) で、foundation はこれを「Zcash ウォレット利用者に影響を与えてきた最近の性能問題に対する潜在的な解決策を提供するかどうかを判断するため」に研究しています。

## よくある間違い

**lightwalletd サーバーがあなたの残高を知っていると思い込むこと。** サーバーはコンパクトブロックを配信するだけであり、それをローカルで復号し解釈するのは、あなた自身の鍵を使うウォレットです。

**同期を早く止めすぎること。** 一部の方法では、完全同期が終わる前でも最近の使用可能な資金を利用できますが、古い履歴やノートの処理はまだ進行中である可能性があります。

**Zcash の同期を透明チェーンの同期と直接比較すること。** 遅い経路は欠陥ではなく、プライバシーを保つためのコストかもしれません。ウォレットは、本来ならパブリックコインのサーバーがあなたのアカウントを公開的に読み取ることで済ませる作業を、自ら行っているのです。
