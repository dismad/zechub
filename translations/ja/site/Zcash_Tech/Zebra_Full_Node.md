## Zebraノードの紹介

Zebra: Rustで構築されたZcashノードインフラストラクチャの革新

Zebraは、Rustで完全に構築された初のZcashノードとして、画期的な成果です。Zcashのピアツーピアネットワークにシームレスに統合され、ネットワークの耐久性を強化する重要なツールとして機能します。取引の検証とブロードキャスト、およびZcashブロックチェーン状態の正確な維持を通じて、Zebraはより分散型なネットワークインフラを実現しています。

## Zcashdノード実装との利点

オリジナルのZcashノードであるzcashdとは異なり、zcashdはBitcoinの基礎となるコードベースに由来し、Electric Coin Companyによって開発されています。一方で、我々の実装は独立した存在として、セキュリティと効率性を重視してからゼロスタートで構築されました。メモリセーフなRust言語の力を利用しています。

異なる起源にもかかわらず、zcashdとZebraは同じプロトコルに従っており、それによりシームレスな通信および相互運用性が可能になります。この革新は、Zcashエコシステムを拡張するだけでなく、ブロックチェーンノード開発の新たな基準も設定しています。

## Zebraランチャーの使用手順

Dockerイメージを使用してZebraを実行することも、または手動で構築することも可能です。システム要件セクションをご覧ください。

### Dockerの使用:

最新リリースを簡単に実行し、先頭に同期するには次のコマンドを実行してください：

```

docker run zfnd/zebra:latest

```

より詳細な手順や詳しい情報を得るためには、[Dockerドキュメント](https://zebra.zfnd.org/user/docker.html)をご覧ください。

### Zebraの構築:

Zebraを構築するにはRust、libclang、およびC++コンパイラが必要です。

- 最新の安定版Rustがインストールされていることを確認してください。Zebraはこれのみでテストされています。
- 必要なビルド依存関係には以下があります：
  - libclang（libclang-devまたはllvm-devとも呼ばれます）
  - clangまたは他のC++コンパイラ（すべてのプラットフォームにg++、macOSではXcode）
  - protoc（Protocol Buffersコンパイラ）は、Protocol Buffers v3.12.0で導入された*--experimental_allow_proto3_optional*フラグとともに使用してください。（2020年5月16日にリリースされました）

### Archでの依存関係:

依存関係が満たされていることを確認した後、以下のコマンドを使用してZebraを構築およびインストールしてください：

```

cargo install --locked zebrad

```

次のように実行してZebraを起動してください：

```
zebrad start

```


## オプションの設定と機能:


### - 設定ファイルの初期化:

  - 次のコマンドを使用して設定ファイルを生成します：
    
  ```
  zebrad generate -o ~/.config/zebrad.toml
  
  ```

  - 生成された*zebrad.toml*はLinuxのデフォルトのプリファレンスディレクトリに配置されます。他のOSのデフォルト位置についてはドキュメントをご覧ください。



### - プログレスバーの設定:

  - *tracing.progress_bar*を*zebrad.toml*で構成して、ターミナルで重要なメトリクスをプログレスバーを使用して表示します。注意: 進行状況バーの推定値が非常に大きくなるという既知の問題があります。



### - マイニングの設定:

  - Dockerで*MENER_ADDRESS*とポートマッピングを指定することで、Zebraはマイニングにカスタマイズできます。詳細については[マイニングサポートドキュメント](https://zebra.zfnd.org/user/mining-docker.html)をご覧ください。



### - カスタムビルド機能:

  - Prometheusメトリクス、Sentryモニタリング、実験的なElasticsearchサポートなどの追加のCargo機能でZebraの機能を拡張できます。

  - 複数の機能はインストール時に`--features`フラグのパラメータとしてリストして組み合わせることができます。


### ノート: リリースビルドでは、いくつかのデバッグおよびモニタリング機能が性能最適化のために無効になっています。

実験的および開発者向け機能の詳細な一覧については、[APIドキュメント](https://docs.rs/zebrad/latest/zebrad/index.html#zebra-feature-flags)をご覧ください。


# Zebraのシステム要件とネットワーク構成

最適なパフォーマンスと信頼性を確保するために、Rustで完全に構築された革新的なZcashノードであるzebradをコンパイルおよび実行するための推奨されるシステム要件は以下の通りです：

### システム要件:
- CPU: 4コア
- RAM: 16 GB
- ディスク容量: 300 GBの空きディスク容量（バイナリのコンパイルおよびキャッシュされたチェーン状態の保存に使用）
- ネットワーク: 毎月少なくとも300 GBのアップロードとダウンロードが可能な100 Mbpsネットワーク接続


Zebraのテストスイートは、あなたのマシン仕様によっては1時間以上かかることがあります。遅いシステムでもZebraをコンパイルおよび実行することは可能ですが、我々はまだテストを通じて正確な性能境界を確立していません。


### ディスク要件:
- Zebraはキャッシュされたメインネットデータに約300 GB、テストネットデータに10 GBを使用します。ディスク使用量は時間が経つにつれて増加する可能性があります。
- データベースは定期的にクリーンアップされ、特にシャットダウンまたは再起動時にデータの整合性を保証しています。強制終了やパニックによって引き起こされた不完全な変更は、Zebraを再起動した際にロールバックされます。


### ネットワーク要件とポート:
- Zebraは次のTCPポートを使用してインバウンドおよびアウトバウンド接続を行います：
  - メインネット: 8233
  - テストネット: 18233
- 特定のlisten_addrでZebraを構成することで、このアドレスがインバウンド接続に広告されるようになります。アウトバウンド接続は同期のために不可欠ですが、インバウンド接続はオプションです。
- Zcash DNSシーダーへのアクセスはOSのDNSリゾルバー（通常ポート53）を通じて必要です。
- Zebraは任意のポートでアウトバウンド接続を確立できますが、zcashdは他のネットワーク上のDDoS攻撃を軽減するためにデフォルトポートでのピアを好む傾向があります。


### 通常のメインネットネットワーク使用:
- 初期同期: 最初の同期には300 GBのダウンロードが必要で、その後のダウンロードでは成長が予想されます。
- 継続的な更新: ユーザーの取引サイズとピアからのリクエストに応じて、毎日のアップロードおよびダウンロードは10 MBから10 GBの範囲になります。
- Zebraは内部データベースバージョンが変更されるたびに初期同期を開始し、バージョンアップ時にフルチェーンのダウンロードが必要になる可能性があります。
- ラウンドトリップ遅延が2秒以下のピアが好ましいです。この閾値を超える場合は、サポートチームにチケットを提出してください。


これらの推奨事項と構成に従うことで、Zcashネットワーク内でZebraの効率性および有効性を最大化できます。問題が発生したり、さらなる支援が必要な場合は、サポートチームがすぐにご指導いたします。
