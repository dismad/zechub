### Crosslink Protocol

#### **はじめに：ZcashのハイブリッドPoSとCrosslink Protocol**

Crosslink Protocolは、Zcashの進化における画期的な開発であり、Zcashを**Hybrid Proof-of-Stake (PoS)** と **Proof-of-Work (PoW)** のモデルへと導くものです。従来のPoWは、ネットワークの安全性を確保するうえで信頼性が高い一方で、エネルギー消費や、大規模な産業型マイニングに伴う中央集権化リスクについて批判も受けています。Crosslinkは、実証済みのPoWの堅牢性と、PoSの効率性およびガバナンス上の利点を組み合わせたハイブリッドシステムを導入します。

![image](https://github.com/user-attachments/assets/a2ffb19d-e570-4723-b669-a66e14fc6b71)

この移行は、ブロックチェーンのイノベーションにおける世界的な潮流とも一致しており、多くのプロジェクトが環境的に持続可能で分散化されたメカニズムへ移行しています。Crosslinkのデュアル・コンセンサス・モデルにより、Zcashは強力な暗号学的プライバシー保証を維持しつつ、現代的な課題に対応する形で進化できます。

このHybrid Proof-of-Stake (PoS) アプローチは、従来のProof-of-Work (PoW) とPoSを組み合わせ、51%攻撃のような脆弱性に対処しつつ、分散性を維持し、エネルギー消費を削減することを目指しています。Hybrid PoSでは、ステークされたZECに基づいてブロックを検証する公証人が導入されます。この仕組みは、チェーンのセキュリティとチェックポイント検証を向上させ、純粋なPoWシステムに対するより堅牢な代替案を提供するよう設計されています​。

なぜ最初のテストとしてHybrid PoS/PoWなのか？
純粋なPoSへの前進となるため
マイニングとステーキングのユースケースおよびエコシステム間のクロスオーバーを同時に可能にするため。
PoSプロトコルがより大きなバリデータ・ステークと信頼を得るまで、起こりうるセキュリティ上の問題を軽減するため。
この一般的なアプローチは、Ethereumによって本番環境で実証されています

### CROSSLINK
Crosslink protocolは、ZcashのハイブリッドProof-of-Work/Proof-of-Stake (PoW/PoS) 段階のために提案された設計です。これはPoWをByzantine Fault Tolerance (BFT) プロトコルと統合し、PoWまたはPoSのいずれか一方が安全である限り、確実なファイナリティを実現します。この設計は、マイナーの参加を維持しながら、ステークによる検証を取り入れることで、ネットワークのセキュリティと分散性を強化することを目的としています。Crosslink 2と呼ばれる提案の重要な特徴は、BFTの提案者とマイナーを統合することでアーキテクチャを簡素化している点です。この合理化されたアプローチは、構造的変更を最小限に抑え、「ダミー」のBFTレイヤーを利用可能にすることで、高いセキュリティ基準を維持しつつ、プロトタイプ作成とデプロイを容易にします。

実装計画には、Crosslink 2* をZcashのZebraクライアントに統合するための推定エンジニアリングコストを含むロードマップが含まれています。この段階的なデプロイは、ステークホルダーのインセンティブの均衡を取り、混乱を減らし、スケーラビリティ、使いやすさ、分散化に関するZcashの目標と整合させることに重点を置いています。プロトコルの堅牢なセキュリティ特性に対する信頼が高まっていることも、Zcashの進化における重要な一歩としての可能性をさらに裏付けています。エネルギー効率への対応とコンセンサスメカニズムの強化によって、Crosslinkは進化し続けるブロックチェーンの課題に対する前向きな解決策を提示します。詳細については、[GitHub repository](https://github.com/ShieldedLabs/crosslink-deployment) と [Zcash Community Forum](https://forum.zcashcommunity.com) を参照してください。

#### **Crosslinkの狙いと目的**

Crosslink Protocolは、Zcashの将来にとって重要ないくつかの戦略目標に対処するよう設計されています。

1. **分散化**:
   - PoSを取り入れることで、Zcashは特化型PoWハードウェア（ASIC）への依存を減らします。こうしたハードウェアは、しばしば少数の大規模事業者にマイニング能力を集中させます。
   - PoSは、コイン保有者が自らの資産をステークしてネットワークを保護することで、より幅広いコミュニティの参加を可能にし、より分散したコンセンサスを実現します。
   - ステークによる検証を導入することで、このプロトコルは経済的参加者がコンセンサスにおいて積極的な役割を果たすことを保証し、マイニングだけへの依存を減らします。

2. **強化されたガバナンス**:
   - コイン保有者はステーキングを通じて投票権を得て、ネットワークのアップグレード、資金配分、エコシステムの優先事項に関する意思決定に影響を与えられるようになります。この民主的な仕組みは、プロトコルの進化をコミュニティの利益と一致させます。

3. **エネルギー効率**:
   - 部分的にPoSへ移行することで、エネルギー需要が大幅に低下し、Zcashは世界的な持続可能性イニシアチブと足並みをそろえます。PoSは、計算負荷の大きいPoWと比べて、本質的にリソース集約度が低い仕組みです。ハイブリッドシステムは、高いセキュリティを維持しながら、PoW専用システムと比較してエネルギー使用量を削減することを目指しています​

4. **経済的セキュリティと持続可能性**:
   - PoWとPoSを組み合わせることで、ネットワーク参加者に対する経済的インセンティブが多様化し、単一の仕組みに過度に依存することなく、強固なセキュリティを確保します。
   - また、ステーキングは参加者に予測可能な報酬モデルをもたらし、長期投資家にとって魅力的な提案となります。
 
5. セキュリティの向上: Crosslinkは、PoWに加えてPoSを統合することで、チェーン再編成攻撃に対するネットワークの耐性を高めることを目指しています。


### Crosslinkのセキュリティおよびパフォーマンス目標

Crosslink protocolは、Zcashに対して2種類の台帳、すなわち**finalized ledger (LOG_fin)** と **lower-latency ledger (LOG_ba)** を提供することを目指しています。finalized ledgerは、Byzantine Fault Tolerance (BFT) またはblockchain (BC) プロトコルのいずれかについて合理的な前提が成り立つ限り、ロールバック安全性を保証します。これは、ネットワーク分断下でもライブ性とセキュリティを維持するよう設計されており、同等のブロック承認数に対して、現在のZcashブロックチェーンの2倍をわずかに超える程度のレイテンシしか増えません。

lower-latency ledgerは、finalized ledgerを最大でも *L* ブロックまでしか先行しません。これはblockchain protocol単独のもとでロールバック安全性を保証し、既存のZcashモデルより悪化しないレイテンシとセキュリティを維持します。合理化されたCrosslink 2* の設計では、lower latency ledgerはPoWチェーンとして機能することで、開発と採用を簡素化します。

### 制限付き可用性とSafety Mode

Crosslinkには、lower-latency ledgerがfinalized ledgerを大きく先行してしまうことに伴うリスクに対処するための**Safety Mode**が組み込まれています。これにより、不均衡なアカウント状態や、サービス提供者による一時的な対策における未検証のセキュリティギャップといった不整合を防ぎます。Safety Modeは、finalized ledgerが定数 *L* ブロックを超えて遅れた場合に有効化されます。この状態では、ブロックチェーンはPoWの動作を継続し（基本的なセキュリティを確保しつつ）、問題が解決されるまで経済活動は一時停止されます。この仕組みは、大規模な攻撃のような例外的状況からの回復を可能にしつつ、ガバナンスに基づくロールバック方針を支えるよう設計されています。

#### **PoWマイナーの収益への影響**

Crosslinkは、Zcash初期の発展においてPoWマイナーが果たしてきた基盤的な役割を認識しつつ、段階的な移行に備えています。

- **ブロック報酬の減少**:
   - 時間の経過とともに、PoSバリデータが報酬のより大きな割合を受け取るようになり、PoWマイナーの収益は減少します。この再配分は、ハイブリッドモデルにおけるPoWの役割が縮小していくことを反映しています。
   
- **公正な移行**:
   - このプロトコルは変更を段階的に導入することで、マイナーが適応したり、ステーキングへの移行や他のネットワークサービスへの貢献といった、Zcashエコシステム内の新たな役割を模索したりするための十分な時間を確保します。

- **中央集権化リスクの軽減**:
   - PoSステーキングプールは権力集中を防ぐよう設計されており、より小規模な参加者にも対等な条件で参加する機会を提供します。この包摂的なアプローチは、現在のASICベースのマイニングで見られる集中化に対抗するものです。

- ブロック報酬の一部がPoSバリデータに再配分されるため、PoWマイナーの収益は減少します。この再配分により、マイナーとステーカーの双方にネットワーク保護への報酬を与える、バランスの取れたインセンティブシステムが実現されます。
- マイナーへの経済的影響を軽減しつつ、ステークホルダーの参加を促進するため、段階的な移行が計画されています​

#### **技術的詳細とデプロイ**

Crosslink Protocolは、Shielded LabsがZodlのような主要なエコシステムパートナーと連携しながら、積極的に開発およびデプロイを進めています。プロトコルの実装には以下が含まれます。
- PoS参加者のための安全なステーキングメカニズムの確立。
- マイナーとステーカーの間でインセンティブの均衡を取るための報酬構造の変更。
- 移行期間中の後方互換性とシームレスなユーザー体験の確保。
- 公証人システム: このプロトコルには、ブロックに署名する公証人が組み込まれています。初期段階では固定公証人が使用され、その後、ステークされたZECに基づいて公証人が選出される動的システムへ移行します。​
- アクティベーション・ロジック: Crosslinkの導入には、ステーク分配プロセスの定義や、ハイブリッド・コンセンサスを支えるためのネットワーク・プロトコル・ルールの更新を含む、Zcashコンセンサスルールの変更が必要です​
- 段階的デプロイ: ネットワークの安定性とコミュニティの適応を確保するため、このプロトコルは段階的に展開されます。初期段階では技術的実装に重点を置き、その後、公証人選出のためのガバナンス統合が続きます​。

技術的な詳細を確認し、進捗を追跡するには、[GitHub上のCrosslink Deployment Repository](https://github.com/ShieldedLabs/crosslink-deployment) を参照してください。
