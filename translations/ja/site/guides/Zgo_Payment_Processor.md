# ZGo決済処理：Zcashを預金なしで受け取る

ZGoは、Zcash用の非預金型決済処理です。顧客が自分のウォレットからZECを支払うと、ZGoはZcashブロックチェーンを監視し、資金をシールドされた転送を通じて直接商人のウォレットに届けます。ZGoはその間、お金を保持することはありません。

このガイドでは、決済フローがどのように機能するか、アカウントの設定方法、XeroおよびWooCommerceとZGoを統合する方法について説明します。また、初めてのセットアップでよくある2つの間違いも取り上げます。

## なぜZGoを使うのか

ほとんどの暗号資産決済処理は預金型です。資金はまず処理業者のアカウントに届き、その後商人に転送されるため、第三者が一時的に資金を管理し、凍結、遅延、または報告することがあります。

ZGoは逆のアプローチを取ります。支払いは顧客のウォレットから直接商人のウォレットへ、Zcashシールドされた取引を通じて移動します。処理業者は請求書を生成し、ブロックチェーンを監視するだけです。中間の残高や引き出しフロー、または決済を妨げる第三者は存在しません。

商人にとってこれは実用上3つの意味を持ちます：入金したZECに関する完全な管理権、デフォルトでシールドされた取引によるプライバシー、および中央集約型のプロバイダーがオンラインまたは健全であることを依存しないこと。

## ZGoの仕組み

ZGoを単独で使用するか、XeroやWooCommerceを通じて使用するかに関係なく、支払いフローは同じです：

1. 商人はZGoで支払いリクエストを生成し、金額、請求書ID、およびZcash受信アドレスが含まれたQRコードとして表示されます。
2. 顧客はZcashウォレット（WordPressプラグインではOrchard、Sapling、および透明アドレスタイプすべてサポート）でQRコードをスキャンし、支払いを承認します。
3. 取引は顧客のウォレットから商人のウォレットへのシールドされた転送としてZcashネットワークにブロードキャストされます。
4. ZGoはZcashブロックチェーンで取引を監視します。
5. 5回の確認後、ZGoは支払いを完了とマークし、接続された統合（Xero、WooCommerce、またはウェブフック）に通知します。

5回の確認というしきい値が鍵です。それ以前は支払い進行中のものであり、支払い済みとはみなされません。注文の履行、在庫更新、および商人側での不可逆的なアクションはステップ5を待つ必要があります。

ZGoはデスクトップまたはモバイルの現代的なブラウザで動作し、どちらの側にもインストール不要です。顧客にはZcashウォレットが必要であり、商人にはZcashウォレットとZGoアカウントが必要です。

## アカウントの設定

ZGoアカウントを作成するには、少量のZECを持つZcashウォレットが必要です。この少量のZEC残高はアカウント初期化取引のオンチェーン手数料をカバーします。どの主要なZcashウォレットでも使用可能ですが、現在のオプションについては[ZecHub Wallets](https://zechub.wiki/wallets)をご覧ください。

基本的な設定:

1. ブラウザで[zgo.cash](https://zgo.cash/)を開きます。
2. 商人の管理下にあるZcashウォレットを使用してアカウントを作成します。このウォレットは鍵を保持する必要があります。取引所の預金アドレスは動作しません（[よくある間違い](#common-mistakes)をご覧ください）。
3. 初期化トランザクションを送信してウォレットを確認します。
4. 受け取りアドレスを設定します。このアカウントを通じて処理されたすべての支払いはこのウォレットに届きます。

アカウントがアクティブになると、同じ商人はワンタイム決済（ポップアップイベントでの単一QRコード）またはXeroやWooCommerceを通じた永続的な設定にZGoを使用できます。

## XeroとZGo

[Xero](https://www.xero.com/)は多くの中小企業が使用するクラウド会計プラットフォームです。ZGo–Xero統合により、商人はXeroで請求書を作成し、顧客がZECで支払いを行い、取引が確認されるとXeroが自動的に請求書を支払済みとしてマークできます。

その仕組み:

1. 商人は通常通りXeroで請求書を作成します。
2. ZGoは請求書にZcash支払いオプションを添付します。
3. 顧客は自分のウォレットからZECで支払いを行います。
4. ZGoは[Zcashブロックチェーン](https://z.cash/)で取引を監視します。
5. 5回の確認後、ZGoは支払いをXeroに報告し、請求書が決済済みとしてマークされます。

ZECは商人のウォレットに届き、ZGoまたはXeroが管理するアカウントには届きません。Xero内の会計記録はオンチェーン決済と自動的に同期されます。

初めてのセットアップでは専用のウォークスルーを参照してください：[Xero統合設定](https://hedgedoc.vergara.tech/s/4iXC67fmb)。

## WooCommerceとZGo

[WooCommerce](https://woocommerce.com/)および[WordPress](https://wordpress.org/)上でのオンラインショップでは、ZGoは専用のプラグインを提供しています。このプラグインはチェックアウト時にZcashを支払い方法として追加し、支払いが確認されると注文状態を自動的に処理します。

<img width="672" height="378" alt="ZGo WooCommerceプラグインチェックアウトおよび注文フロー" src="https://github.com/user-attachments/assets/55a791bb-1947-4f55-b5b9-55083be8ed49" />

WooCommerceストア内でのエンドツーエンドフロー:

1. 顧客はチェックアウトに到達し、Zcashを支払い方法として選択します。
2. プラグインが支払いリクエストを生成し、チェックアウトページでQRコードを表示します。
3. 顧客は自分のウォレットから支払います。
4. 取引はZcashネットワークにブロードキャストされ、ZGoが始めて監視を開始します。
5. 5回の確認後、ZGoがプラグインに対して支払いが完了したと報告します。
6. プラグインはWooCommerce注文を支払済みとしてマークし、注文データベースを更新します。

ステップ6が完了するまで注文は支払済みとはみなされません。以前の状態（ブロードキャスト、最初の確認）は顧客に「支払い済み、確認待ち」と表示できますが、在庫、履行、および下流の自動化は最終状態を待つ必要があります。

このプラグインはWordPress内に管理ダッシュボードもインストールし、商人は通常のWooCommerce注文ビューと並行して注文や入金したZEC支払いを監視できます。プラグインは現在のすべてのZcashアドレスタイプ（Orchard、Sapling、およびTransparent）をサポートしており、どの準拠ウォレットからでも顧客が取引を完了できます。

## 機能

**非預金型。** 支払いはシールドされた取引を通じて顧客のウォレットから直接商人のウォレットに移動します。ZGoはその間、資金を保持することなく、商人は常に完全な管理権を持ち続けます。

**柔軟な展開。** ZGoはポップアップ市場での一時的な使用、永続的なポイントオブセールスの設定、またはXeroやWooCommerce統合を通じてオンラインストアのバックエンドとして使用できます。

**ブラウザベース。** 顧客側も商人側もインストール不要です。ZGoはデスクトップまたはモバイルの現代的なブラウザで動作します。

**ウォレット互換性。** Orchard、Sapling、およびTransparentアドレスタイプをサポートする主要なZcashウォレットは、顧客側に追加設定なしでZGo請求書を支払うことができます。

**統合。** Xero（会計）とWooCommerce（電子商取引）との直接統合により、最も一般的な商人ワークフローが箱の外からカバーされています。

## よくある間違い

**5回の確認完了前に注文を支払済みとして扱う。** ブロードキャストされたトランザクションは確認済みの支払いとは異なります。トランザクションがまだ確認されなかったり、置き換えられたりする可能性があります。5回の確認完了後のみZGoが支払いを最終的に報告し、その時点で注文を下流で支払済みとしてマークすべきです。商人がブロードキャストイベントに在庫または履行をトリガー設定している場合、詐欺的または失敗した支払いは実際の損失を引き起こします。

**ZGoに取引所の預金アドレスを指す。** Zcashアドレスのように見えるかもしれませんが、取引所の預金アドレスは取引所によって管理され、商人ではありません。取引所が鍵を保持しているため、資金も保持しており、非預金型処理を使用する理由を無効にします。ZGoで設定されたウォレットアドレスは、商人自身がシードフレーズを直接管理できるウォレットである必要があります。

**ZGoをウォレットとして扱う。** ZGoは支払い処理であり、ウォレットではありません。鍵を保存したり、残高を保持したり、商人に資金を支出させたりしません。ZGoがルーティングする資金を受け取るために、商人の管理下にある別のZcashウォレットが必要です。

## まとめ

ZGoは、預金を放棄することなく、中間者に依存することなく、公開チェーン上でのトランザクション詳細を暴露することなく、Zcash支払いを受け取る方法を商人に提供します。XeroおよびWooCommerceとの2つの統合により、最も一般的な商人ワークフローがカバーされており、それ以外の場合は、任意のブラウザからZGoを単独で使用できます。

セットアップは簡単です：Zcashウォレットを取得し、[zgo.cash](https://zgo.cash/)でアカウントを作成し、支払いリクエストを直接生成するか、関連する統合をインストールします。
