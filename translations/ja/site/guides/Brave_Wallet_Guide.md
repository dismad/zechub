# Brave WalletでシールドされたZcash (ZEC) を送受信する方法：ステップごとのプライバシーガイド

Brave Walletは、Braveブラウザに直接組み込まれた、安全なネイティブ暗号資産ウォレットです。拡張機能は不要です。


## シールドされたZECのサポートが重要な理由


Zcashは、シールドされたトランザクションによってデフォルトでプライバシーを提供する、数少ない暗号資産の1つです。

Brave Walletは現在、OrchardのシールドされたZECをサポートしており、サードパーティ製ソフトウェアなしでプライベートな取引を可能にしています。

ユーザーは、残高やメタデータを明かすことなく、Braveブラウザ内ですべて完結した形でZECを送受信できるようになりました。

## パート1：Brave Walletのセットアップ

**このパートで扱う手順：**

1. Braveブラウザを開く（現時点ではデスクトップ版のみ）

2. Walletアイコンをクリックするか、brave://wallet に移動する

3. ウォレットをセットアップする（新規作成またはインポート）

4. 暗号資産ダッシュボードにアクセスする


![img1](https://github.com/user-attachments/assets/f54cd1a1-8569-4925-ba1c-7597d030593e)


## パート2：Brave WalletにZcash (ZEC) を追加する

**このパートで扱う手順：**

1. Walletダッシュボードで「Manage Assets」をクリックする

2. ZECを検索して有効にする

3. これでウォレットにZECが表示される


![img2](https://github.com/user-attachments/assets/6f2b2190-cf55-4394-9d5f-29ff9b5bb525)


## パート3：シールドされたZECを受け取る（Unified Address）

**このパートで扱う手順：**

1. 資産一覧でZECをクリックする

2. 「Receive」を選択する

3. BraveがUnified Address (UA) を生成する。これは、シールドされたトランザクションと透明なトランザクションの両方をサポートする1つのアドレスです

4. QRコードをコピーするかスキャンする

5. 送信者に自分のUAを共有する



![img3](https://github.com/user-attachments/assets/53c940b6-1a03-4fa7-aefa-d3478f678a88)



## パート4：シールドされたZECをプライベートに送信する

**このパートで扱う手順：**

1. ZEC -> Send を選択する

2. 受取人のUnified Addressまたはシールドアドレスを貼り付ける

3. 金額を入力する

4. トランザクションの詳細を確認する

5. 「Send」をクリックする。これで資金はOrchardプールを使用して、プライベートに送金されます


![img4](https://raw.githubusercontent.com/Kellyjoe8/zechub/refs/heads/main/Internet_20250808_172118_4.webp)




## アドレスの種類

以下は、各Zcashアドレスの種類の説明です。

**t-addr (t1...)** - TransparentアドレスはBitcoinのアドレスのように機能するため、トランザクションはブロックチェーン上で公開されます。Zcash以外のウォレットでも使いやすい一方で、プライバシーは提供されません。

**z-addr (zs...)** - Shieldedアドレスは、Zcashのプライバシー技術（Sprout/Sapling）を使用して、送信者・受信者・金額を隠します。詳細を明らかにできるのは、Viewing Keyまたは所有者のみです。

**UA (u... or QR)** - Unified Addressは、複数のアドレス形式（transparent、shielded、Orchard）を1つにまとめたものです。これにより、可能な限りプライバシーを優先しつつ、最大限の互換性が得られます。
