## Вступ до вузла Zebra

Представляємо Zebra: революція в інфраструктурі вузлів Zcash на Rust

Знайомтеся з Zebra — новаторським досягненням як першим вузлом Zcash, повністю створеним на Rust. Безшовно інтрований у peer-to-peer мережу Zcash, Zebra слугує ключовим інструментом, що зміцнює стійкість мережі. Завдяки своїм основним функціям перевірки та трансляції транзакцій, а також ретельному підтриманню стану блокчейну Zcash, Zebra сприяє більш децентралізованій мережевій інфраструктурі.

## Переваги над реалізацією вузла Zcashd
На відміну від оригінального вузла Zcash, zcashd, який походить від базового кодового дерева Bitcoin і розробляється Electric Coin Company, наша реалізація є автономною сутністю. Zebra розроблено з нуля з акцентом на безпеку та ефективність, використовуючи потужність безпечної щодо пам’яті мови Rust.

Попри різне походження, і zcashd, і Zebra дотримуються одного й того самого протоколу, що забезпечує безперешкодну комунікацію та взаємодію між ними. Ця інновація не лише розширює екосистему Zcash, а й задає новий стандарт для розробки блокчейн-вузлів.

## Інструкції для Zebra Launcher

Ви можете запускати Zebra за допомогою нашого Docker-образу або зібрати його вручну. Будь ласка, перегляньте розділ System Requirements.

### Використання Docker:

Щоб без зусиль запустити наш останній реліз і синхронізувати його до актуального стану мережі, виконайте таку команду:

```

docker run zfnd/zebra:latest

```

Для більш повних інструкцій і детальніших пояснень зверніться до нашої [документації Docker](https://zebra.zfnd.org/user/docker.html).

### Збирання Zebra:

Для збирання Zebra потрібні Rust, libclang і компілятор C++.

- Переконайтеся, що у вас встановлено останню стабільну версію Rust, оскільки Zebra тестується виключно з нею.
- Необхідні залежності для збирання включають:
  - libclang (також відомий як libclang-dev або llvm-dev)
  - clang або інший компілятор C++ (наприклад, g++ для всіх платформ або Xcode для macOS)
  - protoc (компілятор Protocol Buffers) з прапором *--experimental_allow_proto3_optional*, запровадженим у Protocol Buffers v3.12.0 (випущеному 16 травня 2020 року).



### Залежності на Arch:

Після того як ви переконалися, що залежності виконано, переходьте до збирання та встановлення Zebra за допомогою такої команди:

```

cargo install --locked zebrad

```

Запустіть Zebra, виконавши:

```
zebrad start

```


## Додаткові конфігурації та можливості:


### - Ініціалізація файлу конфігурації:

  - Згенеруйте файл конфігурації за допомогою команди:
    
  ```
  zebrad generate -o ~/.config/zebrad.toml
  
  ```

  - Згенерований *zebrad.toml* буде розміщено в типовому каталозі налаштувань Linux. Для типових розташувань в інших ОС зверніться до нашої документації.



### - Налаштування індикаторів прогресу:

  - Налаштуйте *tracing.progress_bar* у вашому *zebrad.toml*, щоб відображати важливі метрики в терміналі за допомогою індикаторів прогресу. Примітка: існує відома проблема, через яку оцінки індикатора прогресу можуть ставати надмірно великими.



### - Налаштування майнінгу:

  - Zebra можна налаштувати для майнінгу, вказавши *MINER_ADDRESS* і зіставлення порту в Docker. Додаткові подробиці можна знайти в нашій [документації щодо підтримки майнінгу](https://zebra.zfnd.org/user/mining-docker.html).


### - Користувацькі можливості збирання:

  - Розширюйте функціональність Zebra додатковими можливостями Cargo, такими як метрики Prometheus, моніторинг Sentry, експериментальна підтримка Elasticsearch тощо.

  - Поєднуйте кілька можливостей, перелічуючи їх як параметри прапора `--features` під час встановлення.


### Примітка: Деякі функції налагодження та моніторингу вимкнені в release-збірках для оптимізації продуктивності.

Щоб переглянути повний список експериментальних можливостей і можливостей для розробників, зверніться до нашої [документації API](https://docs.rs/zebrad/latest/zebrad/index.html#zebra-feature-flags).
 

# Системні вимоги та налаштування мережі для Zebra

Щоб забезпечити оптимальну продуктивність і надійність, ми рекомендуємо такі системні вимоги для компіляції та запуску zebrad — революційного вузла Zcash, повністю створеного на Rust:

### Системні вимоги:
- CPU: 4 ядра CPU
- RAM: 16 ГБ
- Дисковий простір: 300 ГБ вільного місця на диску для компіляції бінарних файлів і зберігання кешованого стану ланцюга
- Мережа: мережеве з’єднання 100 Мбіт/с із мінімум 300 ГБ завантажень і вивантажень на місяць


Будь ласка, зверніть увагу, що виконання тестового набору Zebra може зайняти понад годину залежно від характеристик вашої машини. Хоча повільніші системи можуть бути здатні компілювати та запускати Zebra, ми ще не встановили точні межі продуктивності шляхом тестування.


### Вимоги до диска:
- Zebra використовує приблизно 300 ГБ для кешованих даних Mainnet і 10 ГБ для кешованих даних Testnet. Очікуйте, що використання диска з часом зростатиме.
- База даних регулярно очищується, особливо під час завершення роботи або перезапусків, що забезпечує цілісність даних. Незавершені зміни через примусове завершення або panic відкочуються після перезапуску Zebra.


### Вимоги до мережі та порти:
- Zebra використовує такі TCP-порти для вхідних і вихідних з’єднань:
  - 8233 для Mainnet
  - 18233 для Testnet
- Налаштування Zebra з конкретним listen_addr дає змогу оголошувати цю адресу для вхідних з’єднань. Хоча вихідні з’єднання є необхідними для синхронізації, вхідні з’єднання є необов’язковими.
- Необхідний доступ до DNS seeders Zcash через DNS-резолвер ОС (зазвичай порт 53).
- Хоча Zebra може встановлювати вихідні з’єднання на будь-якому порту, zcashd надає перевагу вузлам на типових портах, щоб зменшити ризик DDoS-атак на інші мережі.


### Типове використання мережі Mainnet:
- Початкова синхронізація: для початкової синхронізації потрібне завантаження 300 ГБ, і очікується зростання обсягів подальших завантажень.
- Поточні оновлення: очікуйте щоденні вивантаження та завантаження в діапазоні від 10 МБ до 10 ГБ залежно від розміру транзакцій користувачів і запитів від вузлів.
- Zebra ініціює початкову синхронізацію при кожній зміні внутрішньої версії бази даних, що потенційно може вимагати повного завантаження ланцюга під час оновлення версії.
- Перевага надається вузлам із затримкою round-trip 2 секунди або менше. Якщо затримка перевищує цей поріг, будь ласка, надішліть запит для отримання допомоги.


Дотримуючись цих рекомендацій і налаштувань, ви зможете максимізувати ефективність і результативність Zebra в мережі Zcash. Якщо у вас виникнуть будь-які проблеми або вам знадобиться додаткова допомога, наша команда підтримки завжди готова надати рекомендації.
