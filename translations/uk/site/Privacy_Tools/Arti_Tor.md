# **Arti: Tor-клієнт нового покоління на Rust**
![Atri Logo](https://gitlab.torproject.org/uploads/-/system/project/avatar/647/sliced-onions.png)

**Arti** — це ініціатива Tor Project зі створення клієнта **Tor** нового покоління з використанням мови програмування **Rust**. Arti розроблено як модульний, вбудовуваний і готовий до продакшену інструмент, що забезпечує безпечнішу та ефективнішу реалізацію протоколів анонімності **Tor**. У **Arti version 1.4.0** було представлено кілька важливих оновлень:

- **новий RPC-інтерфейс** для розширеної взаємодії.
- Підготовчу роботу для **підтримки relay**.
- Покращення **стійкості onion service на боці сервісу до атак типу denial-of-service**.

Цей реліз продовжує зусилля Tor Project із надання користувачам і розробникам Tor кращої безпеки, продуктивності та модульності.


---


## **Встановлення клієнта Arti**

Виконайте ці кроки, щоб встановити та запустити **Arti** як SOCKS-проксі у вашій системі.

---

### **Крок 1: Налаштуйте середовище розробки Rust**

Перш ніж ви зможете зібрати Arti з вихідного коду, вам потрібно встановити останню стабільну версію **Rust**.

#### Щоб встановити Rust:

1. Відвідайте офіційний [сайт Rust](https://www.rust-lang.org/).
2. Дотримуйтесь інструкцій зі встановлення для вашої операційної системи.
3. Перевірте встановлення, виконавши:
   
   ```sh
   rustc --version
   ```

Це підтвердить, що у вашій системі встановлено останню стабільну версію Rust.

#### **Примітка для користувачів Windows**:
- Rust можна встановити у Windows через [**Rustup**](https://rustup.rs/) — інсталятор toolchain. Також переконайтеся, що ви налаштували сумісне середовище збірки (у Windows вам може знадобитися **Visual Studio Build Tools**).
  
---

### **Крок 2: Клонуйте репозиторій Arti**

Щоб отримати найновішу версію клієнта Arti, вам потрібно буде клонувати репозиторій з [**GitLab**](https://gitlab.torproject.org/tpo/core/arti).

#### Кроки:
1. Відкрийте термінал (Command Prompt, PowerShell або Git Bash у Windows).
2. Виконайте таку команду, щоб клонувати репозиторій:
   
   ```sh
   git clone https://gitlab.torproject.org/tpo/core/arti.git
   ```
4. Перейдіть до щойно створеного каталогу *arti*:
   
   ```sh
   cd arti
   ```

Це завантажить вихідний код Arti на ваш локальний комп’ютер.

---

### **Крок 3: Зберіть бінарний файл Arti**

Після клонування репозиторію вам потрібно зібрати Arti за допомогою **Cargo**, менеджера пакетів і інструмента збірки для Rust.

#### Щоб зібрати Arti:
1. У терміналі виконайте таку команду:
   ```sh
   cargo build --release
   ```

Ця команда компілює код Arti та оптимізує його для продакшену (прапорець *--release*). Бінарний файл буде створено в каталозі *target/release*.

#### Розташування скомпільованого бінарного файлу:
- Після збірки бінарний файл Arti буде розташовано тут:  
  ```sh
  target/release/arti
  ```

Ви можете запускати цей бінарний файл безпосередньо з термінала.

---

### **Крок 4: Запустіть SOCKS-проксі Arti**

Щоб використовувати Arti як SOCKS-проксі (який маршрутизуватиме ваш інтернет-трафік через мережу Tor), потрібно запустити проксі.

#### Щоб запустити SOCKS-проксі:
1. Виконайте таку команду:
   ```sh
   ./target/release/arti proxy -p 9150
   ```

Ця команда запускає Arti як **SOCKS5-проксі** на **порту 9150**, який є типовим портом, що використовується Tor для SOCKS-трафіку.

---

### **Крок 5: Налаштуйте застосунки для використання Arti**

Коли Arti вже запущено як SOCKS-проксі, вам потрібно налаштувати ваші застосунки на його використання для маршрутизації трафіку через мережу Tor.

#### Кроки:
1. У налаштуваннях вашого застосунку (наприклад, веббраузера або термінальної програми) знайдіть **налаштування проксі**.
2. Встановіть **SOCKS5-проксі** на *localhost:9150*.

Це маршрутизуватиме весь трафік з ваших застосунків через **мережу Tor**, використовуючи Arti як посередника.

---

## **Інтеграція Arti з мережею Tor**

Ось спрощена діаграма, яка показує, як Arti працює разом із мережею Tor:


```plaintext
[Application] --(SOCKS5)--> [Arti SOCKS Proxy] --(Tor Protocol)--> [Tor Network]
```

- **Application** підключається до **Arti SOCKS Proxy** за допомогою протоколу **SOCKS5**.
- Далі Arti взаємодіє з **мережею Tor**, забезпечуючи анонімізацію вашого трафіку під час проходження через мережу.

## **Репозиторій GitLab і внесок у проєкт**

Якщо ви зацікавлені у внеску в розробку **Arti**, ви можете переглянути код і долучитися через **GitLab**.

- **Посилання на репозиторій**: [GitLab-репозиторій Arti](https://gitlab.torproject.org/tpo/core/arti)
- **Клонувати репозиторій**:
  ```sh
  git clone https://gitlab.torproject.org/tpo/core/arti.git
  ```

### **Форк і внесок у проєкт**:
1. **Створіть fork** репозиторію в GitLab (потрібен обліковий запис GitLab).
2. Прив’яжіть ваш fork репозиторію до локального середовища:
   ```sh
   git remote add _name_ git@gitlab.torproject.org:_name_/arti.git
   git fetch _name_
   ```
   Замініть *_name_* на ваше ім’я користувача GitLab.

3. **Надішліть зміни** у ваш fork:
   ```sh
   git push _name_ main
   ```

4. **Створіть Merge Request (MR)** у GitLab:
   Перейдіть до розділу Merge Request у вашому GitLab fork:
   ```plaintext
   https://gitlab.torproject.org/_name_/arti/-/merge_requests
   ```

### **Рекомендації щодо Merge Request**:
- **Не виконуйте rebase і squash комітів під час review**.
- За потреби використовуйте *fixup!* або *squash!* для автоматичного злиття комітів.
- Намагайтеся **додавати нові коміти**, а не зливати їх під час циклу review.

### **Додаткові примітки**:

- **Готові бінарні файли**: Станом на зараз **Arti** не надає офіційних готових бінарних файлів. Ви повинні зібрати клієнт із вихідного коду, як описано вище.
- **Знання Rust**: Якщо ви долучаєтеся до розробки Arti, зверніть увагу, що кодова база все ще розвивається, і в міру додавання нових функцій можливі зміни або рефакторинг.
