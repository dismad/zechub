# Zcash Testnet

## Що таке Zcash Testnet?

**Zcash Testnet** — це паралельний блокчейн до реальної основної мережі Zcash (Mainnet), який відтворює той самий протокол, правила та логіку транзакцій — але з двома ключовими відмінностями:

1. **Монети не мають реальної грошової вартості** — вони називаються **TAZ**, а не ZEC, і використовуються лише для тестування.  
2. **Оновлення мережі, інструменти та програмне забезпечення спочатку тестуються тут**, перш ніж їх розгортають у реальному блокчейні Zcash.  

Іншими словами, Testnet — це своєрідна **пісочниця або експериментальне середовище**, де розробники, аудитори та творці можуть випробовувати ідеї, не ризикуючи реальними грошима.


## Чому існує Testnet?

Testnet є критично важливим для розробки блокчейнів, тому що **реальні блокчейни, такі як Zcash, є незмінними** — щойно транзакції підтверджено в основній мережі, їх неможливо скасувати. Testnet надає **безпечну копію** для експериментів, тестування та налагодження функцій перед розгортанням у Mainnet.

### Використання Testnet

#### 1. Розробка та інтеграція програмного забезпечення

Розробники, які створюють гаманці, біржі, майнінгове ПЗ або інструменти приватності, можуть безпечно тестувати їх у Testnet. Можливості включають:

- Надсилання та отримання транзакцій  
- Майнінг нових блоків за допомогою монет TAZ із нульовою вартістю  
- Створення користувацьких інтерфейсів і API  
- Тестування функцій приватності транзакцій (transparent vs shielded)  

**Приклад:**  
Такі інструменти, як [`zcash_tx_tool`](https://github.com/QED-it/zcash_tx_tool), використовують Testnet для створення транзакцій і тестування функціональності shielded asset у Zcash.  

**Сценарій із реального світу:**  
Розробник гаманця може підключити програмне забезпечення до RPC-ендпойнта Testnet і змоделювати повний життєвий цикл — створення адрес, надсилання shielded транзакцій і перевірку балансів — перед запуском у Mainnet.

#### 2. Тестування оновлень мережі

Zcash періодично оновлює свій базовий протокол (наприклад, Nu5, Nu6). У Testnet нові оновлення активуються **раніше, ніж у Mainnet**, що дає змогу розробникам і спільноті виявляти та виправляти помилки.

**Приклад:**  
Нове правило консенсусу або новий тип транзакцій спочатку запускають у Testnet. Після успішного тестування він активується в Mainnet на заздалегідь визначеній висоті блоку.

#### 3. Тестування реалізацій вузлів

Zcash підтримує кілька реалізацій програмного забезпечення вузлів — `zcashd` і **Zebra** (вузол на Rust, який підтримує Zcash Foundation). Testnet дає змогу тестувати вузли в реальних умовах без фінансового ризику.  

Розробники вузлів можуть:

- Перевіряти поширення блоків  
- Тестувати RPC-інтерфейси  
- Спостерігати за поведінкою вузла під навантаженням  
- Тестувати взаємодію з майнінговим ПЗ  

#### 4. Навчання та освіта

Початківці можуть вивчати функції Zcash, такі як майнінг, створення shielded транзакцій і використання Unified Addresses.  
Навчальні матеріали та документація спільноти надають доступ до **кранів Testnet, оглядачів блоків і посібників**.


## Реальні випадки використання Testnet

### 1. Тестування розробником (гаманець / застосунок)

- Підключення до Zcash Testnet  
- Запит TAZ із крана  
- Надсилання shielded транзакцій  
- Перевірка приватності та стабільності UI  

Жоден реальний ZEC не буде втрачено, навіть якщо стануться помилки.

### 2. Тестування інтеграції біржі

- Запуск вузла Testnet  
- Використання ендпойнтів Zebrad JSON-RPC для обробки транзакцій  
- Тестування автоматизованої логіки депозитів/виведень  

Це забезпечує безпечний production-код і запобігає фінансовим втратам.

### 3. Випробування налаштування майнінгу

- Використання шаблонів для майнінгу  
- Тестування валідації блоків  
- Спостереження за винагородами за майнінг (лише TAZ)  
- Налаштування продуктивності майнінгу  

Це запобігає простою або втраті прибутку під час переходу на Mainnet.

### 4. Академічні / протокольні дослідження

Дослідники можуть тестувати інновації, такі як **stateless verification**, **оптимізація доказів з нульовим розголошенням** або інші протокольні експерименти, використовуючи Testnet.  
Досвідчені користувачі також можуть запускати **власні Testnet або середовища regtest** для спеціалізованих експериментів.


## Ключові відмінності між Mainnet і Testnet

| Feature               | Mainnet           | Testnet                  |
|-----------------------|-----------------|--------------------------|
| Вартість монет        | Реальний ZEC     | TAZ (без грошової вартості) |
| Ризик                 | Фінансовий ризик | Безпечно для тестування  |
| Оновлення протоколу   | Продакшн         | Рання активація          |
| Винагороди за майнінг | Реальна емісія   | Лише тестова винагорода  |
| Корисність мережі     | Живі транзакції  | Тестування та розробка   |

## Поширені хибні уявлення

- **Монети Testnet щось варті** -> Неправда, TAZ мають нульову вартість.  
- **Втратити монети Testnet — це важливо** -> Неправда, жодна реальна цінність не втрачається.  
- **Testnet і Mainnet ідентичні** -> Неправда, Testnet часто скидається і не має такого економічного захисту, як Mainnet.

## Що таке TAZ?

**TAZ** — це версія монет Zcash для Testnet:  

- Не є реальними грошима; не можуть бути обміняні на ZEC або фіат  
- Використовуються для тестування, розробки та навчання  
- Дотримуються всіх правил Zcash: їх можна надсилати, майнити та використовувати в shielded адресах  

**Приклад:**  
Розробник може надіслати 100 TAZ з однієї адреси Testnet на іншу, щоб протестувати функцію гаманця без ризику втратити реальний ZEC.  

Сприймайте TAZ як **«ігрові гроші» для Zcash Testnet**.


## Що таке крани?

**Кран** — це сервіс, який безкоштовно надає монети TAZ для тестування:

- Зазвичай це вебсайти або API  
- Користувачі надають адресу Testnet; кран надсилає невелику кількість TAZ  
- Це усуває потребу майнити TAZ вручну  

**Приклад:**  
1. Відвідайте кран Testnet (наприклад, [testnet.zecfaucet.com](https://testnet.zecfaucet.com) | [fauzec.com](https://fauzec.com/))  
2. Введіть свою адресу Testnet  
3. Запросіть TAZ  
4. Отримайте TAZ миттєво, щоб почати тестування  

**Чому це важливо:**  
- Безпечне тестування без ризику для ZEC  
- Доступність для початківців і розробників  
- Швидке прототипування для гаманців, бірж і застосунків



## Гаманці Zkool і Zingo!

### Zkool

- Мультиакаунтний гаманець для просунутих користувачів Zcash  
- Підтримує seed-фрази, viewing keys, transparent і shielded адреси  
- Може підключатися до Mainnet, Testnet або Regtest через повні вузли або сервери lightwallet

### Zingo!

- Мобільний гаманець, орієнтований на приватність і простоту  
- Підтримує shielded і unified адреси  
- Оновлений для підтримки протоколів Testnet (включно з NU6 Testnet)

## Короткий підсумок

- **Zcash Testnet** — це безпечне середовище-пісочниця для створення, тестування та експериментів  
- Випадки використання: тестування розробниками, тестування вузлів, інтеграція бірж, дослідження та освіта  
- Замість ZEC використовуються **монети TAZ**, які не мають реальної вартості  
- Testnet є необхідним перед розгортанням функцій у Mainnet
