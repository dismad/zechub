- Використовуйте shielded-адреси для зберігання спільних коштів
- Обмежуйте видимість балансів і транзакцій
- Відокремлюйте операційні ролі від публічної особистості
- Використовуйте memo для внутрішньої звітності

Кошти спільнот у публічних блокчейнах розкривають:
- Загальний баланс скарбниці
- Історію витрат
- Особи учасників

Це створює ризики:
- Націлювання на кошти
- Внутрішні конфлікти через тиск прозорості
- Втрата стратегічної приватності

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-lock.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="toolbox icon"/> Чому Zcash?

Zcash дозволяє спільнотам:
- Приватно зберігати кошти
- Виконувати транзакції, не розкриваючи деталей
- Підтримувати внутрішню звітність без публічного розголошення

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-toolbox.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="toolbox icon"/> Що вам потрібно

- Гаманець Zcash (або кілька гаманців для розділення ролей)
- Чіткі внутрішні процеси для управління коштами
- За бажанням: спільна документація для відстеження

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="step icon"/> Покроковий посібник

### Крок 1: Створіть shielded-адресу скарбниці

- Згенеруйте **z-address** для скарбниці
- Вона стане основним рахунком для зберігання коштів


### Крок 2: Визначте ролі

Розділіть обов’язки:
- Скарбник (контролює кошти)
- Учасники (запитують витрати)
- Рецензенти (внутрішньо схвалюють)

Уникайте пов’язування цих ролей із публічними особистостями.



### Крок 3: Отримуйте кошти приватно

- Приймайте внески через shielded-адресу
- Заохочуйте учасників надсилати кошти з shielded-гаманців



### Крок 4: Відстежуйте транзакції внутрішньо

Використовуйте:
- Memo у гаманці
- Off-chain журнали (наприклад, спільні документи)

Приклад memo: 
 - Грант #12 - Виплата розробнику



### Крок 5: Виконуйте платежі приватно

- Надсилайте кошти через shielded-транзакції (z → z)
- Уникайте публічного розкриття особистостей одержувачів
