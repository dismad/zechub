## Коротко

- Отримуйте платежі через shielded-адреси
- Розділяйте ідентичності для кожного клієнта (за потреби)
- Використовуйте memo для внутрішнього обліку
- Зберігайте свій дохід приватним

- Свій дохід

- Свою фінансову історію

Це може призвести до:
- Втрати переговорної переваги
- Ризиків для приватності
- Небажаної видимості

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="step icon"/> Покрокова інструкція

### Крок 1: Створіть адресу для платежів

Згенеруйте shielded (z) адресу.

За бажанням:
- Використовуйте різні адреси для різних клієнтів


### Крок 2: Поділіться адресою з клієнтом

Надішліть свою z-адресу безпосередньо клієнту.

Уникайте:
- Публікації її у відкритому доступі, якщо в цьому немає потреби

### Крок 3: Використовуйте memo для відстеження

Під час отримання платежів:
- Попросіть клієнтів додавати memo (наприклад, ID рахунку)

Це допоможе вам:
- Відстежувати платежі, не розкриваючи дані публічно


### Крок 4: Підтримуйте розділення ідентичностей

За потреби:
- Використовуйте окремі гаманці або адреси для кожного клієнта
- Уникайте пов’язування ідентичностей між собою

<br/>

- Використання transparent-адрес для платежів

- Публічне пов’язування гаманця з особою  
- Відсутність обліку платежів (що призводить до плутанини)
