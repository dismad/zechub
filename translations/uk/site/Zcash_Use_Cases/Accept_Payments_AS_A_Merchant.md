## Коротко

- Приймайте платежі, використовуючи shielded-адресу
- Уникайте розкриття даних про транзакції клієнтів
- Використовуйте memo або позаланцюгові системи для відстеження замовлень
- Зберігайте конфіденційність доходів бізнесу

Традиційні криптоплатежі розкривають:
- Поведінку клієнтів під час покупок
- Дохід продавця
- Історію транзакцій

Це призводить до:
- Втрати приватності клієнтів
- Витоку конкурентної інформації
- Фінансової прозорості, якої ви можете не хотіти

Zcash надає:
- Приватні платежі для клієнтів
- Конфіденційне відстеження доходів
- Захист бізнес-активності

- Простий платіжний процес (ручний або інтегрований)
- Необов’язково: система відстеження замовлень

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="step icon"/> Покрокова інструкція

### Крок 1: Налаштуйте платіжну адресу

- Згенеруйте shielded (z) адресу
- Використовуйте її для отримання платежів


### Крок 2: Показуйте платіжні інструкції

Для кожної транзакції:
- Показуйте свою z-адресу
- Вказуйте суму платежу
- За бажанням додавайте посилання на memo

### Крок 3: Відстежуйте замовлення

Використовуйте:
- Memo (наприклад, ID замовлення)
- Або позаланцюгове відстеження (база даних, електронна таблиця)


Приклад:
- Замовлення #4821


### Крок 4: Підтверджуйте платіж

- Відстежуйте вхідні shielded-транзакції
- Зіставляйте memo або суму із замовленням


### Крок 5: Дотримуйтеся правил приватності

- За можливості уникайте повторного публічного використання адрес
- Не розкривайте історію транзакцій
- За потреби використовуйте окремі гаманці для різних підрозділів бізнесу
