# Known Issues

## NixOS / Nix-specific

### Android SDK from Nix store is read-only
Gradle/Tauri may fail when trying to write to the SDK directory. The `devenv.nix` sets `ANDROID_USER_HOME` and `GRADLE_USER_HOME` to writable paths under `$DEVENV_STATE` to work around this.

### `NDK_HOME` must be exported
`tauri android init` fails silently if `NDK_HOME` is unset. The `devenv.nix` computes it dynamically from the NDK path installed by the Android module.

### SDK license acceptance
The `android.extraLicenses` list in `devenv.nix` must include all required licenses or `sdkmanager` (and by extension Tauri) will refuse to install packages.

### rust-overlay overlay module
The devenv config imports `rust-overlay` via `inputs.rust-overlay.overlaysModules.default`. If this changes upstream, adjust the import path.

### 16KB page size (NDK r28+)
Android 15+ requires 16KB page alignment. NDK r28+ (pulled by nixpkgs-unstable) produces compliant binaries. If using an older NDK, binaries will be rejected on Android 15+ devices.

## Tauri v2

### `tauri android init` requires clean project state
If `src-tauri/gen/android` already exists, re-running `init` may fail. Delete it first: `rm -rf src-tauri/gen/android`.

### Dev server must be network-reachable for physical device
Tauri replaces `localhost` with the host IP, but on NixOS with a headless VM you may need to set `TAURI_DEV_HOST` explicitly to the VM's network-reachable IP.

### `usesCleartextTraffic` for dev server
The dev server runs on HTTP. The Android manifest generated by `android init` should include `android:usesCleartextTraffic="true"` for debug builds. If the phone cannot connect, check `src-tauri/gen/android/app/src/main/AndroidManifest.xml`.

## Browser-only development

### No Tauri runtime in browser
The `invoke()` call throws when Tauri is absent. The `backend.ts` module wraps all calls with a `window.__TAURI_INTERNALS__` check and falls back to mock data. If a new command is added without a mock handler, it will throw.

### `tauri-plugin-store` not available in browser
Settings persistence falls back to in-memory mock state. Real disk persistence only works in the Tauri webview or on device.

## Android / adb

### Wireless debugging requires same subnet
The phone and VM must be on the same network. If using a VPN or different VLANs, `adb connect` may fail.

### `adb install` may fail on older Android versions
If the device is running Android < 5.0, `adb install -r` may need different flags. The debug APK targets API 34 (Android 14).

## Version bumps

### Updating Tauri v2
Tauri v2 is under active development. When bumping `tauri` and `tauri-plugin-store` in `Cargo.toml`, also update `@tauri-apps/api` and `@tauri-apps/plugin-store` in `package.json` to matching versions.
